BiblioFuse에서 LTR과 RTL 읽기 방향 전환하기

2026년 6월 22일

BiblioFuse에서 왼쪽-오른쪽과 오른쪽-왼쪽 읽기 방향 전환하기

대부분의 서양 만화는 왼쪽에서 오른쪽으로 읽습니다: 오른쪽으로 스와이프하여 페이지를 넘기고, 패널은 왼쪽 상단부터 시작하며 이야기가 일반 책처럼 전개됩니다. 일본 만화는 모든 것이 반대입니다——패널이 오른쪽에서 왼쪽으로 흐르고, 첫 페이지는 권의 오른쪽에 있으며 왼쪽으로 스와이프하여 앞으로 넘깁니다. 두 형식을 모두 읽는다면, 시리즈를 바꿀 때마다 다시 설정할 필요 없이 두 가지를 모두 처리할 수 있는 리더가 필요합니다.

BiblioFuse는 읽기 방향을 책별 설정으로 저장하므로, 만화 라이브러리는 오른쪽에서 왼쪽으로, 서양 만화는 왼쪽에서 오른쪽으로 읽힙니다——세션 사이에 수동 전환 없이.

읽기 방향이 중요한 이유

LTR 모드에서 만화를 열면 이야기를 거꾸로 읽게 됩니다——오른쪽에서 왼쪽으로 흘러야 할 패널이 왼쪽에서 오른쪽으로 표시되어 리듬과 개그를 망칩니다.

BiblioFuse에서 읽기 방향 설정하기

앱 전체 기본값

설정 → 읽기 → 읽기 방향을 열어 전역 기본값을 설정합니다. 옵션은:

  • 왼쪽에서 오른쪽 (LTR) ——서양 만화와 대부분의 EPUB 전자책의 표준
  • 오른쪽에서 왼쪽 (RTL) ——만화와 일부 만화의 표준

가장 많이 읽는 형식을 선택하세요.

책별 재정의

전역 기본값은 새로 열리는 파일에 적용되지만, 전역 설정을 변경하지 않고 책별로 재정의할 수 있습니다.

읽는 중에 화면 중앙을 탭하여 리더 컨트롤을 표시한 다음, 상단 도구 모음의 톱니바퀴 아이콘을 탭합니다. 읽기 방향 토글이 빠른 설정 패널 상단에 나타납니다. 변경은 즉시 적용됩니다.

BiblioFuse는 이 설정을 파일별로 저장합니다. 다음에 같은 책을 열면 마지막으로 설정한 방향으로 열립니다.

방향 전환 시 변경되는 것

스와이프 방향 ——LTR에서는 왼쪽으로 스와이프하면 다음 페이지로 이동. RTL에서는 오른쪽으로 스와이프하면 앞으로 이동. 이 반전은 처음에는 어색하지만 만화 페이지를 물리적으로 왼쪽으로 넘기는 동작을 반영합니다.

두 페이지 레이아웃 순서 ——LTR 가로 모드에서는 왼쪽 페이지가 낮은 번호. RTL에서는 오른쪽 페이지가 낮은 번호로, 인쇄된 만화 권의 구조와 일치합니다.

썸네일 스크롤러 방향 ——리더 하단의 썸네일 바가 반전됩니다. RTL 모드에서 가장 왼쪽 썸네일이 권의 마지막 페이지입니다.

세로 스크롤과 읽기 방향

세로 스크롤 모드(웹툰용)는 LTR/RTL 설정의 영향을 받지 않습니다. 연속 스크롤에는 좌우 페이지 순서가 없습니다.

자주 묻는 질문

RTL로 전환한 후 스와이프 방향이 반대로 느껴집니다. 정상인가요?

네——RTL 모드에서는 오른쪽으로 스와이프하면 다음 페이지로 이동합니다. 만화 페이지를 물리적으로 왼쪽으로 넘기는 동작을 반영합니다. 몇 분이면 익숙해집니다.

읽기 방향이 iPhone과 iPad 사이에서 동기화되나요?

파일별 방향 설정은 읽기 위치와 함께 iCloud를 통해 동기화됩니다. 전역 기본 설정은 기기 로컬이며 동기화되지 않습니다.

결론

BiblioFuse에서 읽기 방향을 전환하는 데 세 번의 탭이면 충분하며, 책별로 자동 저장됩니다. 가장 많이 읽는 형식으로 전역 기본값을 설정하고, 나머지는 책별로 재정의하면 리더가 스와이프 방향, 두 페이지 순서, 썸네일 방향을 추가 구성 없이 처리합니다.
