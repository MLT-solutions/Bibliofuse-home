클라우드 계정 없이 독서 진행 상황 동기화하는 방법

2026년 6월 3일

클라우드 계정 없이 읽기 진행 상황 동기화하는 방법

iPhone에서 망가 시리즈를 읽다가 iPad를 들면 다시 첫 페이지로 돌아가 있습니다. 클라우드 동기화로 해결할 수 있지만, 동기화 서비스에 로그인하고, 라이브러리 메타데이터를 제3자에게 맡기고, 연결이 불안정하면 진행 상황이 틀어질 수도 있다는 점을 감수해야 합니다.

그러고 싶지 않다면? 여행 중 안정적인 연결이 없거나, 클라우드 서비스에 읽기 데이터를 맡기고 싶지 않거나, 그냥 모든 것을 로컬에 보관하고 싶다면?

BiblioFuse는 클라우드 계정에 의존하지 않고 여러 기기 간에 읽기 진행 상황을 일치시키는 방법을 여러 가지 제공합니다. 각 옵션이 어떻게 동작하는지 설명합니다.

읽기 진행 동기화가 생각보다 어려운 이유

얼핏 "진행 상황 동기화"는 간단해 보입니다 — 숫자 하나만 저장하면 되지 않나요. 하지만 만화 라이브러리에는 수십, 수백 권이 있고 각 권마다 총 페이지 수와 마지막으로 읽은 페이지가 다릅니다. 읽기 진행 상황은 책별, 기기별 상태로 여러 기기에서 동시에 변할 수 있습니다. 중앙 조정자(클라우드 서버) 없이는 두 기기가 모두 오프라인 상태에서 서로 충돌하는 업데이트를 만들 수 있습니다.

로컬 전용 동기화 전략은 모두 절충이 필요합니다. 엄격한 기본/복제본 모델을 적용하거나, "마지막으로 쓰인 진행 상황이 이긴다"를 받아들이거나, 같은 네트워크에 있을 때만 동기화하는 방식 중 하나입니다.

BiblioFuse의 iCloud 동기화는 이 조정을 자동으로 처리합니다. 하지만 iCloud가 선택지가 아니라면, 다음 방법들이 다양한 자동화 수준에서 제어권을 줍니다.

옵션 1: Mac에서 Wi-Fi 라이브러리 동기화 (가정 사용에 권장)

Mac에서 BiblioFuse의 Mac 홈 라이브러리 기능을 실행 중이라면 iPhone이 Wi-Fi로 Mac에 연결할 수 있습니다. Mac이 파일과 진행 상황 모두의 신뢰할 수 있는 출처 역할을 합니다.

작동 방식:

  1. Mac에서 BiblioFuse를 열고 설정 → 홈 라이브러리에서 라이브러리 서버를 활성화합니다.
  2. iPhone의 라이브러리 탭에서 네트워크 아이콘을 탭하고 Mac을 선택합니다.
  3. iPhone에 Mac의 라이브러리가 표시되고 Mac의 저장소에서 직접 책을 읽습니다 — iPhone으로 복사 불필요.

진행 상황은 Mac 쪽에 저장됩니다. Mac 라이브러리에서 읽는 기기가 무엇이든 진행 상황은 거기에 기록됩니다. iPhone을 열고 Mac에 다시 연결하면 지난번 Mac 세션에서 멈춘 곳부터 이어집니다.

제한 사항: 두 기기가 같은 Wi-Fi 네트워크에 있어야 합니다. 가정 사용에 가장 적합하며, 여행 중 독서에는 VPN이나 Tailscale을 통한 원격 접속 설정이 필요합니다.

옵션 2: 수동 진행 상황 내보내기 및 가져오기

BiblioFuse에서 읽기 진행 상황을 JSON 파일로 내보내고 다른 기기에서 가져올 수 있습니다. 네트워크 없이 완전히 오프라인으로 사용할 수 있는 가장 휴대하기 좋은 옵션입니다.

기기 A에서 내보내기:

  1. 설정 → 라이브러리 → 진행 상황 내보내기로 이동합니다.
  2. BiblioFuse가 라이브러리의 모든 책에서 마지막으로 읽은 페이지를 담은 .json 파일을 저장합니다.
  3. AirDrop, 파일 앱 또는 다른 방법으로 파일을 공유합니다 — 인터넷 불필요.

기기 B에서 가져오기:

  1. .json 파일을 받고 BiblioFuse로 열거나, 설정 → 라이브러리 → 진행 상황 가져오기로 이동합니다.
  2. BiblioFuse가 가져온 진행 상황을 기존 로컬 진행 상황과 병합하며, 각 책에서 더 높은 페이지 번호를 채택합니다(절대 뒤로 가지 않음).

이 방법은 완전 오프라인입니다. 기기가 가까이 있으면 AirDrop 전송이 거의 즉시 완료됩니다. 유일한 수동 단계는 기기를 전환하기 전에 내보내는 것을 기억하는 것 — 며칠이면 습관이 됩니다.

옵션 3: 로컬 네트워크 드라이브 또는 USB를 활용한 파일 기반 동기화

파일 앱으로 라이브러리를 관리한다면 진행 상황을 책 파일과 함께 이동할 수 있습니다. BiblioFuse는 클라우드 ID가 아닌 파일 이름별로 진행 상황을 저장합니다. 이는 다음을 의미합니다:

  • 같은 파일 이름으로 기기 A에서 기기 B로 파일을 복사하면, 기기 B의 BiblioFuse는 진행 상황 없이 표시합니다(그 기기에는 새 파일이므로).
  • 하지만 책 파일과 진행 상황 내보내기(옵션 2) 파일 둘 다 기기 B에 복사하고 진행 상황 파일을 가져오면 올바른 페이지에서 재개됩니다.

NAS나 공유 SMB 드라이브에서 라이브러리를 관리하는 독자에게 이 방식은 깔끔하게 작동합니다. 책은 드라이브에 남아 있고, 기기는 로컬 네트워크에 있을 때 연결하며, 진행 상황은 수동으로 진행 상황 파일을 동기화할 때까지 각 기기에서 독립적으로 추적됩니다.

옵션 4: 클라우드 동기화 계정 없이 iCloud Drive 사용

자주 혼동을 일으키므로 명확히 짚어두겠습니다. iCloud Drive는 BiblioFuse의 iCloud 동기화와 같지 않습니다.

BiblioFuse의 iCloud 동기화는 NSUbiquitousKeyValueStore와 CloudKit을 사용해 읽기 진행 메타데이터를 동기화합니다. 이를 위해서는 활성 iCloud 계정이 필요합니다.

하지만 BiblioFuse의 iCloud 진행 상황 동기화를 활성화하지 않고도 책 파일을 iCloud Drive(파일 앱)에 저장할 수 있습니다. 이 설정에서:

  • 같은 Apple ID로 로그인된 모든 기기에서 책 파일에 접근 가능
  • 읽기 진행 상황은 기기별 로컬에서 추적되며 동기화되지 않음
  • 진행 상황을 일치시키려면 여전히 옵션 1, 2 또는 3을 사용해야 함

iCloud Drive를 순전히 파일 저장소로만 사용하고 진행 상황 동기화가 필요 없다면, 이는 깔끔한 설정입니다.

옵션 5: 기기별 진행 상황을 받아들이고 북마크 사용하기

모든 독자에게 완벽한 진행 상황 동기화가 필요한 건 아닙니다. 한 번에 한 권씩 읽고 권 중간에 기기를 전환하지 않는다면, 기기별 진행 상황으로 충분합니다.

BiblioFuse에서는 어느 페이지에서든 북마크를 설정할 수 있습니다. 북마크는 이름이 붙은 저장 위치입니다 — 기기를 전환하기 전에 하나 만들어두면 다른 기기에서 바로 점프할 수 있습니다. 설정 한 번 탭, 이동 한 번 탭이면 되므로 가끔 기기를 전환하는 경우 전체 동기화보다 빠릅니다.

올바른 옵션 선택하기

상황최적 옵션
가정에서 독서, Mac 보유Wi-Fi 홈 라이브러리 (옵션 1)
여행 중, 오프라인, iOS 기기 두 대AirDrop을 통한 수동 내보내기/가져오기 (옵션 2)
NAS 또는 공유 드라이브 라이브러리파일 기반 동기화 + 진행 상황 내보내기 (옵션 3)
가끔 기기 전환, 한 번에 한 권북마크 (옵션 5)
Apple ID는 있지만 제3자 클라우드 불사용파일용 iCloud Drive + 수동 진행 상황 동기화

프라이버시 관점

BiblioFuse는 읽기 진행 상황을 기기 로컬에 저장합니다. iCloud 동기화를 비활성화하면 어떤 읽기 데이터도 기기를 벗어나지 않습니다. 프라이버시상의 이유로든 단순한 취향으로든 외부 서비스에 읽기 습관을 저장하고 싶지 않은 독자에게, 로컬 전용 모드는 후속 옵션이 아닌 최우선 모드입니다.

위의 옵션들은 빠진 기능의 우회책이 아닙니다. 클라우드 동기화가 적합하지 않은 상황에서 BiblioFuse를 사용하는 원래 방법입니다.

시작하기

  1. App Store에서 BiblioFuse를 다운로드합니다.
  2. 파일 앱, Wi-Fi 전송 또는 Mac 홈 라이브러리를 통해 만화 또는 전자책 파일을 가져옵니다.
  3. 독서 습관에 맞는 동기화 전략을 선택하거나 조합하세요. 옵션 2와 옵션 5는 어떤 다른 방법과도 함께 사용할 수 있습니다.

클라우드 계정을 거부한다고 편의성을 포기할 필요는 없습니다. 작은 습관 하나 — 기기를 전환하기 전 빠른 AirDrop 내보내기 — 로 읽기 데이터를 누구에게도 맡기지 않고 라이브러리 상태를 일관되게 유지할 수 있습니다.
