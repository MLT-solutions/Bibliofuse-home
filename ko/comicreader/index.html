라이브러리

모든 형식을 위한 하나의 책장.

EPUB, PDF, CBZ, CBR, RAR, ZIP, TXT를 폴더, 검색, 최근 책이 있는 하나의 라이브러리에서 읽을 수 있습니다. 표지 썸네일은 실제 저작권 표지를 사이트 스크린샷에 사용하지 않으면서도 CBZ, CBR, EPUB, PDF 라이브러리를 시각적으로 정리해 줍니다. 파일 앱, 문서 선택기 또는 iCloud Drive에서 가져올 수 있습니다.

  • 태그 및 평점 - 원하는 경우 폴더 없이 정리.
  • 읽기 진행률 - 각 책이 중단 위치를 기억.
  • iCloud 크로스 디바이스 - iPad에서 중단한 곳을 iPhone에서 이어서.

Mac과 PC에서 BiblioFuse는 큰 컬렉션을 위한 데스크톱 라이브러리 허브가 됩니다. 호스트에서 폴더를 관리한 뒤, 집에서는 Wi‑Fi로, 밖에서는 Tailscale 5G로 iPhone 및 iPad의 BiblioFuse에 바로 스트리밍하세요.

  • 로컬 Wi-Fi 스트리밍 - Mac 또는 PC 소스에서 즉시 탐색.
  • 원격 5G 스트리밍 - Tailscale로 어디서나 홈 라이브러리 열기.
  • 로컬, iCloud, SMB 소스 - 하나의 데스크톱 라이브러리와 이미 보관 중인 모든 폴더.
  • 옵션 PIN - 공유 네트워크에서 스트리밍 접근 잠금.

원격 접근에는 호스트와 iOS 기기가 iCloud에 로그인되어 있고 Tailscale로 연결되어 있어야 합니다. 집과 외부 모두에서 가장 안정적으로 쓰려면 두 기기에서 Tailscale을 켜 두세요.

FAQ

자주 묻는 질문

한 번 구매로 모든 플랫폼을 사용할 수 있나요?

아니요. iOS + Mac은 하나의 유니버설 구매입니다. 한 번의 결제로 iPhone, iPad, Mac, Apple Vision Pro를 포함합니다. Android 리더, PC 리더, PC 도구는 각각 별도로 판매됩니다.

BiblioFuse는 Apple Vision Pro에서 작동하나요?

네. BiblioFuse는 App Store의 네이티브 visionOS 앱으로, 호환 모드의 iPad 앱이 아니라 진짜 공간 윈도우입니다. 시선 추적, 핀치, 마우스, 트랙패드, 키보드를 지원하며 iOS + Mac 유니버설 구매에 추가 비용 없이 포함됩니다.

BiblioFuse는 visionOS에서 어떤 공간 읽기 레이아웃을 지원하나요?

세 가지 공간 라이브러리 레이아웃: 공간 책장(떠 있는 책장에 표지 배치), 그리드 보기(대형 라이브러리용 떠 있는 그리드), 기울어진 벽(평점과 태그 필터가 있는 기울어진 갤러리 벽). Mac이나 PC에서 라이브러리를 스트리밍해 Vision Pro 저장 공간을 그대로 둘 수도 있습니다.

iPhone의 BiblioFuse에 책과 폴더를 어떻게 추가하나요?

BiblioFuse는 파일을 iCloud 동기화 폴더에 저장합니다. 파일 앱(BiblioFuse 폴더에 파일 복사), AirDrop, 또는 내장 Wi-Fi 전송을 통해 책을 추가할 수 있습니다. Mac 또는 PC에서 라이브러리 세션을 열고 모든 브라우저에서 파일을 드래그하면 됩니다.

단계별 사이드로드 가이드
Mac 또는 PC에서 iPhone으로의 스트리밍은 어떻게 작동하나요?

Mac 또는 PC가 로컬 서버를 실행하고 연결 주소를 iCloud Drive에 기록합니다. iPhone이 자동으로 감지합니다. 두 기기 모두 Tailscale이 켜져 있어야 합니다(집과 원격으로 작동). 한 기기만 Tailscale이 켜져 있으면 연결이 실패합니다. 두 기기를 항상 켜두면 가장 일관된 환경을 얻을 수 있습니다.

스트리밍 설정에 필요한 것은 무엇인가요?

Mac 또는 PC와 iPhone 모두 다음이 설정되어 있어야 합니다:

  • 두 기기에서 동일한 iCloud 계정 사용
    • iOS와 Mac에서: 설정 → [이름] → iCloud로 이동하여 BiblioFuse가 활성화되어 있는지 확인
    • PC에서: iCloud가 내장되어 있지 않습니다 — Microsoft Store에서 iCloud for Windows 다운로드
  • 두 기기 모두 Tailscale이 켜져 있어야 합니다
    • 둘 다 켜짐 — 집과 5G 원격에서 작동
    • 혼합 (하나는 켜짐, 다른 하나는 꺼짐) — 연결 실패
    • Tailscale 출구 노드를 활성화할 필요 없음
    • iCloud와 Tailscale 계정은 다른 이메일 주소를 사용할 수 있음
  • 앱을 처음 실행할 때 네트워크 액세스 허용
스트리밍이 올바르게 설정되었지만 여전히 작동하지 않습니다 — 무엇을 시도해야 하나요?

오래된 엔드포인트 캐시가 가장 일반적인 원인입니다. iCloud에 저장된 stream_endpoint.json 파일 중 하나 이상의 타임스탬프가 오래되어 BiblioFuse가 호스트를 검색하지 못하고 있습니다.

  • iCloud에서 오래된 파일 삭제
    • iPhone의 경우: 파일 앱 → iCloud Drive → BiblioFuse(루트)와 BiblioFuse → 문서를 엽니다.
    • Mac의 경우: Finder → iCloud Drive → BiblioFuse(루트)와 BiblioFuse → 문서를 엽니다.
    • 두 위치 모두에서 stream_endpoint.json, stream_endpoint 1.json, stream_endpoint 2.json 등의 파일을 모두 삭제하세요.
    • PC의 경우: iCloud가 이 파일들의 클라우드(온라인 전용) 버전만 표시할 수 있으므로, PC에서의 삭제가 안정적으로 작동하지 않을 수 있습니다 — iPhone 또는 Mac을 사용하세요.
  • 삭제 후 호스트(Mac 또는 PC)와 iPhone 모두에서 BiblioFuse를 다시 열기
    • 호스트 앱이 현재 타임스탬프로 새 엔드포인트 파일을 재생성합니다.
    • iPhone에서 스트리밍 소스 목록으로 이동하면 PC 또는 Mac 이름 오른쪽에 녹색 점이 표시되어 감지되었음을 확인할 수 있습니다.
    • 점이 회색이면 몇 초 더 기다리거나 호스트에서 BiblioFuse를 강제 종료한 후 다시 여세요.
스트리밍에 왜 Tailscale이 필요한가요? 집 Wi-Fi로는 안 되나요?

Tailscale은 iOS가 로컬 네트워크 연결을 처리하는 방식 때문에 필요합니다. 다음은 그 경위입니다:

  • 초기 버전은 Tailscale 없이 집 Wi-Fi에서 작동했습니다
    • 앱은 Bonjour(Apple의 로컬 검색)를 HTTP로 사용했습니다 — 잘 작동했습니다
    • 5G 원격 스트리밍이 추가되자, iOS App Transport Security가 원시 IP 주소에 대한 HTTP 연결을 차단하기 시작하여 LAN 전용 경로가 작동하지 않게 되었습니다
  • Tailscale은 집 Wi-Fi에서도 원격에서와 동일하게 작동합니다
    • 두 기기가 같은 네트워크에 있을 때 Tailscale은 트래픽을 직접 라우팅합니다 — 일반 Wi-Fi 대비 속도 차이가 없습니다
    • 집과 외출 시 항상 Tailscale을 켜두면 모드 전환 없이 일관된 설정을 유지할 수 있습니다
  • Tailscale 없는 LAN 전용 모드는 현재 로드맵에 없습니다
    • Tailscale을 꺼도 앱이 이전에 저장된 Tailscale 주소를 시도하여 연결 실패가 발생할 수 있습니다
    • 신뢰할 수 있는 자동 LAN 폴백을 구축하는 것은 복잡하며 현재 계획에 없습니다
    • 가장 간단한 설정: 항상 두 기기에서 Tailscale을 켜두세요. 추가 설정 없이 집 Wi-Fi와 5G 원격 모두에서 작동합니다
Android가 Mac이나 PC에서 책을 받을 수 없는 이유는 무엇인가요?

Android 앱은 스트리밍 핸드셰이크 메커니즘의 기반인 iCloud 통합이 없는 다른 코드베이스를 사용합니다. Android의 최선의 워크플로우는 Syncthing 또는 유사한 도구를 사용하여 파일을 휴대폰에 동기화한 다음 앱에서 해당 폴더를 라이브러리 소스로 추가하는 것입니다.

읽기 진행도, 북마크, 평점, 태그는 어디에 저장되나요?

책 파일이 저장된 위치에 따라 다릅니다:

  • 로컬 저장소 — 해당 기기에만 저장, 기기 간 동기화 없음
  • iCloud 저장소 — iCloud에 저장, 모든 iOS, Mac, PC 기기에서 자동 동기화
  • SMB / 네트워크 공유 (Mac 또는 PC 스트리밍) — 호스트 기기에 저장
    • iOS가 Mac 스트림 읽기: 진행도는 Mac에 저장
    • iOS가 PC 스트림 읽기: 진행도는 PC에 저장
    • PC ↔ Mac: 독립적으로 저장, 데이터 공유 없음
    • Mac과 PC를 모두 스트리밍 호스트로 사용하면 읽기 데이터가 분리됩니다. 일관성을 위해 하나의 호스트만 사용하세요.
PC Reader와 PC EPUB / CBZ 도구의 차이점은 무엇인가요?

PC Reader는 완전한 독서 앱입니다 — 책을 열고 라이브러리를 관리하고 파일을 압축하고 iPhone으로 스트리밍합니다. PC EPUB 및 CBZ 도구는 리더 없이 압축 및 병합에만 집중하는 더 가볍고 저렴한 앱입니다.

BiblioFuse에서 더 나은 텍스트 음성 변환 목소리를 사용하려면 어떻게 해야 하나요?

고품질 음성은 기기에 내장되어 있지만 별도로 다운로드해야 합니다 — 설치 후 BiblioFuse가 자동으로 인식합니다.

  • iPhone / iPad
    • 설정 → 손쉬운 사용 → 말하기 콘텐츠 → 음성으로 이동
    • 언어를 선택하고 향상됨 또는 프리미엄 음성을 다운로드
    • iOS 16+의 프리미엄 음성은 신경망 품질(Siri 수준)
  • Mac
    • 시스템 설정 → 손쉬운 사용 → 말하기 콘텐츠 → 시스템 음성 → 음성 관리로 이동
    • 언어에 맞는 향상됨 또는 프리미엄 음성 다운로드
  • Windows 10 / Windows 11
    • 설정 → 시간 및 언어 → 언어 및 지역으로 이동
    • 읽기 언어를 추가하거나 설치된 언어의 언어 옵션 열기
    • 언어 기능 아래에서 텍스트 음성 변환 설치
    • BiblioFuse를 재시작한 후 새 TTS 세션 시작
  • Windows 11 자연스러운 음성 (선택 사항)
    • 설정 → 접근성 → 내레이터로 이동
    • 자연스러운 음성 추가 아래에서 원하는 음성 다운로드
    • 참고: Windows는 이러한 음성을 내레이터에만 제공하고 모든 앱에는 제공하지 않을 수 있음
  • 앱 업데이트 불필요 — BiblioFuse는 Windows 또는 Apple TTS 시스템을 통해 노출된 새로 설치된 음성을 자동으로 감지
