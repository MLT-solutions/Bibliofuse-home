Pemindaian Arsip Mendalam

Membuka arsip dan membandingkan halaman di dalamnya

Kebanyakan pencari duplikat hanya membandingkan nama atau ukuran file. Comic Duplicate Scanner membuka setiap file CBZ, CBR, ZIP, atau RAR, mengekstrak thumbnail dari halaman sebenarnya, dan membandingkannya. File bernama comic_E1-5 dan ComiX_E1-10 — nama berbeda, rentang episode berbeda — tetap terdeteksi jika halaman pembukanya cocok.

  • Membuka arsip dan membandingkan gambar halaman — bukan sekadar nama file
  • Ambang batas kemiripan yang dapat dikonfigurasi — disarankan 90%
  • Cache hash membuat pemindaian ulang hampir instan
  • Mendeteksi duplikat di berbagai resolusi dan mode warna — hasil pindaian skala abu-abu 1200px dan pindaian berwarna 3000px dari halaman yang sama tetap ditandai
Berfungsi dengan manga, manhwa, dan manhua
Foto dan Pencocokan Nama File

Foto dan nama file mirip — keduanya tercakup

Selain arsip, deteksi foto duplikat JPG, PNG, HEIC, WEBP, BMP, dan GIF menggunakan algoritme dHash. Pencocokan nama file fuzzy menangkap salinan yang hanya berbeda dalam format — "Spider-Man #001" vs "Spider Man 001" — dengan ambang batas yang dapat dikonfigurasi.

  • Format foto: JPG, PNG, HEIC, WEBP, BMP, GIF
  • Algoritme RapidFuzz dengan penghapusan simbol yang dapat dikonfigurasi
  • Mendukung Unicode — berfungsi dengan Jepang, Korea, Cina, dan set karakter apa pun
Pemilihan Otomatis Cerdas

Satu klik menandai file yang tepat untuk disimpan

Tetapkan aturan — simpan yang Terbesar, Terkecil, Terbaru, Tertua, atau file dengan Halaman Terbanyak — dan Pemilihan Otomatis menandai setiap duplikat dalam satu klik. Tinjau hasilnya sebelum apa pun diubah. Tidak ada yang dipindahkan hingga Anda mengonfirmasi.

  • Lima aturan: Terbesar, Terkecil, Terbaru, Tertua, Halaman Terbanyak
  • Peringatan salinan terakhir mencegah penghapusan total yang tidak disengaja
  • Daftar abaikan permanen untuk pasangan varian yang disengaja
File dipindahkan ke Tempat Sampah — dapat dipulihkan kapan saja
Laporan dan Multi-Folder

Ekspor hasil, bandingkan antar folder

Ekspor pemindaian Anda sebagai CSV untuk spreadsheet atau sebagai laporan HTML lengkap dengan thumbnail, skor kemiripan, dan kolom yang dapat diurutkan. Mode Multi-Folder membandingkan dua folder terpisah — berguna untuk menemukan duplikat antara pustaka yang terorganisir dan folder Unduhan.

  • Laporan HTML dengan thumbnail tertanam dan statistik
  • Ekspor CSV untuk spreadsheet dan pengarsipan
  • Perbandingan multi-folder: Folder A vs Folder B

FAQ

Pertanyaan yang Sering Diajukan

Apa itu perceptual hashing dan mengapa penting?

Perceptual hashing membandingkan struktur visual gambar daripada byte file mentah. Dua file dapat memiliki nama, ukuran, dan metadata yang berbeda — tetapi jika kontennya terlihat sama, perceptual hashing akan mencocokkannya. Pencari duplikat standar gagal begitu file diganti nama atau diekspor ulang; Comic Duplicate Scanner tidak.

Bisakah menemukan duplikat yang telah diganti nama?

Ya. Mengganti nama file tidak mengubah kontennya. Comic Duplicate Scanner membandingkan gambar di dalam setiap arsip, sehingga duplikat terdeteksi terlepas dari nama file.

Apakah penghapusan permanen?

Tidak. File yang ditandai dipindahkan ke Tempat Sampah sistem (macOS) atau Recycle Bin (Windows), tidak dihapus secara permanen. Anda dapat memulihkan file apa pun sebelum mengosongkan Tempat Sampah.

Format file apa yang didukung?

Arsip komik: CBZ, CBR, ZIP, RAR. Foto: JPG, PNG, HEIC, WEBP, BMP, GIF. Pencocokan nama file fuzzy berfungsi dengan jenis file apa pun.

Apakah berfungsi dengan foto, bukan hanya komik?

Ya. Pilih mode Foto untuk memindai file JPG, PNG, HEIC, WEBP, BMP, dan GIF. Berfungsi dengan ekspor perpustakaan foto iPhone, dokumen yang dipindai, dan koleksi seni digital.

Apakah ada uji coba sebelum membeli?

Anda dapat memindai hingga 1.000 file secara gratis selamanya. Uji coba gratis 7 hari di Mac App Store membuka akses tak terbatas, dan pembelian Pro satu kali seharga $19,99 tanpa langganan.

Bisakah mencocokkan pindaian berwarna dengan versi hitam-putih dari konten yang sama?

Ya. Algoritma dHash bekerja berdasarkan gradien luminansi, bukan nilai warna. Pindaian berwarna penuh dan versi skala abu-abu dari halaman yang sama menghasilkan hash yang hampir identik dan akan ditandai sebagai duplikat.

Apakah resolusi gambar memengaruhi deteksi duplikat?

Tidak. dHash mengubah ukuran setiap gambar ke grid tetap yang kecil sebelum membandingkan, sehingga pindaian 1200px dan 3000px dari halaman yang sama menghasilkan nilai hash yang sama. Perbedaan resolusi antar sumber diabaikan sepenuhnya.
