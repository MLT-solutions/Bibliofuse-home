BiblioFuse Reader長時間の読書セッションのために構築。

形式、フォルダ、タグ、評価、進捗、ブックマークなど、本当に使うライブラリ機能に集中したリーダー。iPhone、iPad、Mac、PCに本を置けて、容量が必要なときはデスクトップライブラリをiPhoneへストリーミングできます。

広告なしトラッキングなしデバイス上で処理
ライブラリ

あらゆるフォーマットに対応する一つの本棚。

EPUB、PDF、CBZ、CBR、RAR、ZIP、TXT：フォルダ、検索、最近の本を備えた単一ライブラリで読書。ファイル、ドキュメントピッカー、iCloud Drive からインポート。

  • タグと評価 - 必要に応じてフォルダなしで整理。
  • 読書の進捗 - 各書籍が中断箇所を記憶。
  • iCloud クロスデバイス - iPad で中断した箇所を iPhone で再開。
あなたの読み方で

TTS、自動スクロール、ピークズーム。

長時間のセッションに必要な操作を備えた集中リーダー。EPUB と TXT のテキスト読み上げ。コミックとウェブトゥーンの自動スクロール。位置を失わずにパネルを拡大表示するピークズーム。

  • テキスト読み上げ - EPUB と TXT のハンズフリーナレーション。
  • 自動スクロール - ページ送りまたは連続、画像アーカイブ向けに調整。
  • ピークズーム - ダブルタップで密集パネルと小さな文字を拡大。
デスクトップからモバイルへ

ライブラリハブ。iPhoneへストリーミング。

サードパーティサーバー不要 · サブスクリプション不要

MacとPCでは、BiblioFuseが大きなコレクション向けのデスクトップライブラリハブになります。ホスト側でフォルダを管理し、自宅のWi‑FiまたはTailscale経由の5Gで、iPhone/iPadのBiblioFuseへ直接ストリーミングできます。

  • ローカル Wi-Fi ストリーミング - MacまたはPCのソースからすぐに参照。
  • リモート 5G ストリーミング - Tailscale でどこからでも自宅ライブラリを開く。
  • ローカル、iCloud、SMB ソース - 1つのデスクトップライブラリに、既存のすべてのフォルダを。
  • オプション PIN - 共有ネットワークでのストリーミングアクセスをロック。

リモートアクセスには、ホストとiOSデバイスがiCloudにサインインし、Tailscaleに接続されている必要があります。自宅でも外出先でも安定させるには、両方のデバイスでTailscaleをオンにしておくのがおすすめです。

FAQ

よくある質問

1回の購入ですべてのプラットフォームをカバーしますか？

いいえ。iOS + Mac は1回のユニバーサル購入で、iPhone、iPad、Mac をカバーします。Android Reader、PC Reader、PC ツールはそれぞれ別売りです。

iPhoneのBiblioFuseに本やフォルダを追加するには？

BiblioFuse はファイルをiCloud同期フォルダに保存します。ファイルアプリ（BiblioFuseフォルダにファイルをコピー）、AirDrop、または内蔵Wi-Fi転送で本を追加できます。MacまたはPCからライブラリセッションを開き、任意のブラウザからファイルをドラッグしてください。

ステップバイステップのサイドロードガイド
MacまたはPCからiPhoneへのストリーミングの仕組みは？

MacまたはPCがローカルサーバーを起動し、接続アドレスをiCloud Driveに書き込みます。iPhoneが自動的に検出します。両デバイスはTailscaleをオンにする必要があります（自宅とリモートで使用可能）。一方のデバイスだけTailscaleがオンの場合、接続は失敗します。両方を常にオンにすると最も安定した体験が得られます。

ストリーミングの設定に必要なものは？

MacまたはPCとiPhoneの両方で以下を設定してください：

  • 両デバイスで同じiCloudアカウントにサインイン
    • iOSとMacでは：設定 → [あなたの名前] → iCloudを開き、BiblioFuseが有効になっていることを確認
    • PCでは：iCloudは内蔵されていません — Microsoft StoreからiCloud for Windowsをダウンロード
  • 両デバイスでTailscaleをONにする必要があります
    • 両方ON — 自宅と5Gリモートで使用可能
    • 混在（一方がON、もう一方がOFF）— 接続が失敗
    • Tailscaleの出口ノードを有効にする必要はありません
    • iCloudとTailscaleのアカウントは異なるメールアドレスを使用できます
  • アプリの初回起動時にネットワークアクセスを許可する
ストリーミングは正しく設定されているのに動作しない — 何を試せばいい？

古いエンドポイントキャッシュが最も一般的な原因です。iCloud に保存されている stream_endpoint.json ファイルのひとつ以上のタイムスタンプが古く、BiblioFuse がホストを検出できていません。

  • iCloud から古いファイルを削除する
    • iPhone の場合：ファイルアプリ → iCloud Drive → BiblioFuse（ルート）と BiblioFuse → 書類 を開きます。
    • Mac の場合：Finder → iCloud Drive → BiblioFuse（ルート）と BiblioFuse → 書類 を開きます。
    • 両方の場所で、stream_endpoint.json、stream_endpoint 1.json、stream_endpoint 2.json などのファイルをすべて削除してください。
    • PC の場合：iCloud はこれらのファイルのクラウド版（オンラインのみ）しか表示しないことがあるため、PC からの削除は確実に機能しない場合があります — iPhone または Mac を使用してください。
  • 削除後、ホスト（Mac または PC）と iPhone の両方で BiblioFuse を再起動する
    • ホストアプリが現在のタイムスタンプで新しいエンドポイントファイルを再生成します。
    • iPhone でストリーミングソースリストを開くと、PC または Mac の名前の右側に緑のドットが表示され、検出されたことが確認できます。
    • ドットが灰色の場合は、数秒待つか、ホストの BiblioFuse を強制終了して再起動してください。
なぜストリーミングにTailscaleが必要なのですか？自宅Wi-Fiだけでは使えないのですか？

TailscaleはiOSがローカルネットワーク接続を処理する方法のために必要です。以下がその経緯です：

  • 初期バージョンはTailscaleなしで自宅Wi-Fiで動作していました
    • アプリはBonjour（AppleのローカルディスカバリM）をHTTP経由で使用していました — 正常に動作していました
    • 5Gリモートストリーミングが追加されると、iOS App Transport SecurityがIPアドレスへのHTTP接続をブロックし始め、LANのみのパスが壊れました
  • Tailscaleは自宅Wi-Fiでもリモートと同様に機能します
    • 両デバイスが同じネットワーク上にある場合、Tailscaleはトラフィックを直接ルーティングします — 通常のWi-Fiと比較して速度の低下はありません
    • 自宅でも外出先でもTailscaleをオンにしておくと、モードの切り替えなしに一貫したセットアップが得られます
  • TailscaleなしのLANのみモードは現在のロードマップにはありません
    • Tailscaleをオフにしても、アプリが以前に保存されたTailscaleアドレスを試みてハンドシェイク失敗になる場合があります
    • 信頼性の高い自動LANフォールバックを構築することは複雑で、現時点では予定されていません
    • 最もシンプルな設定：両デバイスで常にTailscaleをオンにしておいてください。追加設定なしで自宅Wi-Fiと5Gリモートの両方に対応します
AndroidがMacやPCから本を受信できないのはなぜですか？

Androidアプリはストリーミングのハンドシェイクメカニズムの基盤となるiCloud統合のない別のコードベースを使用しています。Androidの最善のワークフローは、SyncthingまたはHJようなツールを使ってファイルをスマートフォンに同期し、そのフォルダをアプリのライブラリソースとして追加することです。

読書の進捗、ブックマーク、評価、タグはどこに保存されますか？

本のファイルがどこに保存されているかによって異なります：

  • ローカルストレージ — そのデバイスにのみ保存、デバイス間の同期なし
  • iCloudストレージ — iCloudに保存、すべてのiOS、Mac、PCデバイスで自動同期
  • SMB / ネットワーク共有（MacまたはPCストリーミング）— ホストデバイスに保存
    • iOSがMacストリームを読む：進捗はMacに保存
    • iOSがPCストリームを読む：進捗はPCに保存
    • PC ↔ Mac：独立して保存され、データは共有されない
    • MacとPCの両方をストリーミングホストとして使用すると、読書データが分割されます。一貫性のために1つのホストを使用することをお勧めします。
PC ReaderとPC EPUB / CBZツールの違いは何ですか？

PC Readerは完全な読書アプリです — 本を開き、ライブラリを管理し、ファイルを圧縮し、iPhoneにストリーミングします。PC EPUBとCBZツールは、リーダーなしで圧縮と結合のみに特化した、より軽量で安価なアプリです。

BiblioFuseでより良いテキスト読み上げの声を使うにはどうすればよいですか？

高品質な音声はデバイスに内蔵されていますが、別途ダウンロードが必要です — インストール後、BiblioFuseが自動的に認識します。

  • iPhone / iPad
    • 設定 → アクセシビリティ → スピーチ → 声 に移動
    • 言語を選択し、「拡張」または「プレミアム」の声をダウンロード
    • iOS 16+ のプレミアム音声はニューラルクオリティ（Siriレベル）
  • Mac
    • システム設定 → アクセシビリティ → スピーチ → システムの声 → 声を管理 に移動
    • 言語の「拡張」または「プレミアム」の声をダウンロード
  • Windows 10 / Windows 11
    • 設定 → 時刻と言語 → 言語と地域 に移動
    • 読書言語を追加するか、インストール済み言語の言語オプションを開く
    • 言語機能の下にある「テキスト読み上げ」をインストール
    • BiblioFuseを再起動し、新しいTTSセッションを開始
  • Windows 11 ナチュラルボイス（オプション）
    • 設定 → アクセシビリティ → ナレーター に移動
    • 「ナチュラルボイスの追加」から使いたい音声をダウンロード
    • 注意：Windowsはこれらの音声をナレーターのみに提供し、すべてのアプリには対応しない場合があります
  • アプリのアップデートは不要 — BiblioFuseはWindowsまたはAppleのTTSシステムを通じて公開された新しい音声を自動検出
