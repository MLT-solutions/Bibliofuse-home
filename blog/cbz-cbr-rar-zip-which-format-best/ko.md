# CBZ, CBR, RAR, ZIP — 만화 아카이브 형식 중 어떤 게 최선인가?

다운로드한 만화의 절반은 `.cbz`, 몇 개는 `.cbr`, 하나는 그냥 `.rar`입니다. 리더를 열면 다 읽히긴 하는데, 문득 궁금해집니다. 형식이 중요한 걸까? 전부 하나로 변환해야 할까? 실제로 어떤 차이가 있는 걸까?

주요 만화 아카이브 형식을 쉽게 정리하고, [BiblioFuse](https://bibliofuse.com/ko/)가 모든 형식을 문제없이 읽는 이유를 설명합니다.

## 만화 아카이브 형식의 실체

CBZ, CBR, ZIP, RAR은 모두 만화 전용 형식이 아닙니다. 범용 아카이브 컨테이너일 뿐이고, 내부에는 이미지 파일(주로 JPEG나 PNG, 최근엔 WebP도 많음) 폴더만 들어 있습니다. 만화 리더는 아카이브를 메모리에서 압축 해제한 뒤 순서대로 이미지를 표시합니다.

즉, 형식 선택은 거의 전적으로 **압축 알고리즘**과 **호환성**의 문제이고, 만화를 위한 특별한 기능과는 관계없습니다.

## CBZ — 대부분의 컬렉션에서 명확한 승자

CBZ는 ZIP 파일의 확장자를 `.cbz`로 바꾼 것뿐입니다. 그게 전부입니다. CBZ를 압축 해제하면 이미지 폴더가 나옵니다.

**CBZ가 기본 형식으로 최선인 이유:**

- ZIP은 오픈·무료 형식으로 모든 운영체제와 아카이브 도구가 기본 지원
- macOS, Windows, Linux 모두 별도 소프트웨어 없이 ZIP 파일을 열 수 있음
- 읽기/쓰기 구현이 쉬워 사실상 모든 만화 리더가 CBZ 지원
- RAR과 달리 특허나 라이선스 문제 없음
- 전체 아카이브를 재압축하지 않고도 페이지 추가·삭제·순서 변경 가능

**압축 효율 트레이드오프:** ZIP은 Deflate 압축을 사용해 빠르지만 최고 효율은 아닙니다. 이미 압축된 JPEG 이미지가 담긴 만화 아카이브에서는 RAR과의 파일 크기 차이가 보통 5% 미만입니다. PNG 중심 아카이브라면 RAR이 10~20% 작을 수 있지만, 근본 원인은 PNG 자체의 크기입니다.

**결론:** 컬렉션을 새로 만든다면 CBZ를 사용하세요.

## CBR — 호환성은 좋지만 기술적으로 열등

CBR은 RAR 파일의 확장자를 `.cbr`로 바꾼 것입니다. RAR은 WinRAR의 개발자 Eugene Roshal이 만든 독점 형식으로 ZIP보다 약간 나은 압축률을 제공하며, 특히 PNG 같은 비압축 이미지 형식에 효과적입니다.

**CBR의 문제점:**

- RAR은 독점 형식입니다. RAR 파일을 만들려면 WinRAR이나 Unix용 RAR의 라이선스 버전이 필요
- 오픈소스 UnRAR 라이브러리는 읽기만 가능하고 생성 불가
- RAR 5에서 새 헤더 형식이 도입되어 일부 구형 리더가 처리 불가
- CBR 재패킹이나 편집에는 별도 도구가 필요

**CBR을 만나는 경우:** 대부분의 CBR 파일은 WinRAR이 Windows의 주류 압축 프로그램이었던 2000년대 초~중반에 만들어진 것입니다. CBZ 대비 실질적인 장점이 없어 신규 릴리스에서 CBR은 거의 사용되지 않습니다.

**CBR을 CBZ로 변환해야 할까요?** 저장 공간이 부족하고 PNG 중심의 CBR이 많다면 변환으로 공간을 절약할 수 있습니다. JPEG 중심 컬렉션은 크기 차이가 미미해 변환할 가치가 없습니다. [BiblioFuse](https://bibliofuse.com/ko/)는 CBR을 기본 지원하므로 읽기 위해 변환할 필요가 없습니다.

## RAR과 ZIP — 기본 형식

일반 `.rar`과 `.zip` 파일은 만화 리더에서 CBR, CBZ와 동일하게 동작합니다. 차이는 확장자뿐입니다. 일부 스캔 도구는 `.cbz`나 `.cbr`로 이름을 바꾸지 않고 원본 아카이브 확장자 그대로 저장합니다. BiblioFuse iOS 버전은 `.zip`과 `.rar` 파일을 직접 지원하므로 이름 변경이 필요 없습니다.

## CB7과 CBT — 마이너 형식

**CB7**은 7-Zip 파일(`.7z`)을 `.cb7`로 바꾼 것입니다. 7-Zip의 LZMA2 압축은 텍스트 같은 압축 가능한 데이터에서 ZIP·RAR보다 훨씬 뛰어나지만, 만화는 대부분 이미 압축된 JPEG 이미지라 실제 절약 효과는 작고 호환 리더도 적습니다.

**CBT**는 TAR 파일을 `.cbt`로 바꾼 것으로, 압축 없이 파일을 묶기만 합니다. Linux 출처 아카이브에서 가끔 등장합니다. 두 형식 모두 보급률이 낮으니 특별한 이유 없이는 CBZ를 사용하세요.

## PDF — 아카이브가 아니지만 흔한 형식

PDF는 문서 형식이지 이미지 아카이브가 아닙니다. PDF로 저장된 만화는 이미지가 내장되어 있지만 레이아웃 오버헤드가 추가되고 페이지 단위 편집에는 전문 도구가 필요합니다. 출판사의 디지털 판매 만화와 스캔 단행본에서 흔합니다. BiblioFuse는 PDF를 읽을 수 있지만, 이미지 기반 PDF의 읽기 경험은 동등한 CBZ보다 떨어집니다 — 탐색이 느리고, 페이지별 메타데이터가 없으며, 이미지당 파일 크기도 큽니다.

만화 PDF를 CBZ로 변환하고 싶다면 [BiblioFuse 웹 도구](https://bibliofuse.com/ko/webapp/)를 이용하세요. 브라우저에서 완전히 실행되며 업로드가 필요 없습니다.

## 형식 비교표

| 형식 | 컨테이너 | 압축 | 자유 생성 | 범용 지원 | 적합한 경우 |
|------|---------|------|----------|---------|-----------|
| CBZ | ZIP | Deflate | ✅ 가능 | ✅ 가능 | 신규 컬렉션, 스캔 |
| CBR | RAR | RAR/RAR5 | ⚠️ WinRAR 필요 | ✅ 대부분 | 구형 아카이브 |
| ZIP | ZIP | Deflate | ✅ 가능 | ✅ 가능 | 일반 ZIP 지원 리더 |
| RAR | RAR | RAR/RAR5 | ⚠️ WinRAR 필요 | ✅ 대부분 | 구형 아카이브 |
| CB7 | 7-Zip | LZMA2 | ✅ 가능 | ⚠️ 일부 | 고압축 실험 |
| CBT | TAR | 없음 | ✅ 가능 | ⚠️ 일부 | Linux 출처 아카이브 |
| PDF | PDF | 다양 | ✅ 가능 | ✅ 가능 | 출판사 DRM 프리 작품 |

## BiblioFuse의 모든 형식 처리 방식

[BiblioFuse(iPhone 및 iPad용)](https://bibliofuse.com/ko/)는 CBZ, CBR, ZIP, RAR, CB7, CBT, EPUB, PDF를 기본 지원합니다. 가져오기 전에 컬렉션을 변환할 필요가 없습니다.

모든 처리는 기기 내에서 이루어집니다. 서버에 파일이 업로드되지 않습니다. 대규모 다형식 라이브러리도 BiblioFuse가 시리즈별로 정리하고 iCloud 동기화로 권별 읽기 진행 상황을 추적합니다 — 기본 형식이 CBZ든 CBR이든 상관없습니다.

BiblioFuse의 도구 탭에서 iPhone에서 직접 형식을 변환할 수 있습니다. CBR 아카이브를 선택하고 변환을 탭한 뒤 CBZ를 선택하면, 앱 안에서 재패킹이 완료됩니다.

## 실용적 권장사항

**새 컬렉션을 처음부터 만드는 경우:** CBZ를 사용하세요. 오픈 형식이고, 범용 호환되며, 다루기 쉽습니다.

**기존 CBR 컬렉션:** 특별한 이유(디스크 공간, CBR을 지원하지 않는 도구)가 없다면 변환하지 않아도 됩니다. CBR은 BiblioFuse와 모든 현대 리더에서 정상 동작합니다.

**혼합 형식 라이브러리:** 보기 좋자고 굳이 통일할 필요 없습니다. BiblioFuse처럼 여러 형식을 처리하는 리더가 있으면 단일 형식 컬렉션을 유지할 필요가 없습니다.

**출판사 PDF 만화:** 편집이나 권 합치기가 필요하지 않으면 PDF 그대로 두세요. 필요하다면 BiblioFuse 웹 도구로 CBZ로 변환하는 것이 합리적입니다.

형식 논쟁은 모든 형식을 다루는 리더를 갖추는 것보다 훨씬 덜 중요합니다. 그게 있으면 컨테이너는 투명해집니다.
