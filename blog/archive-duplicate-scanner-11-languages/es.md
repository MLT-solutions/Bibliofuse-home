## El Problema con la Detección de Duplicados por Nombre de Archivo

Los buscadores de duplicados estándar comparan nombres, tamaños o hashes. Ese enfoque falla con archivos de cómics porque:

- **Mismo contenido, nombre diferente**: `Spider-Man_001.cbz` y `spiderman_issue01.cbz` son el mismo cómic, pero ninguna comparación de nombres lo detecta
- **Rangos de episodios superpuestos**: `manga_vol1_ch1-5.cbz` y `manga_vol1_ch3-8.cbz` contienen capítulos idénticos dentro de diferentes archivos
- **Archivos recodificados**: un CBZ reexportado tiene un hash diferente pero contenido de páginas idéntico

[Comic Duplicate Scanner](/es/archive/) resuelve esto extrayendo una muestra de imágenes de página de cada archivo y comparándolas visualmente mediante hashing perceptual.

## Cómo Funciona la Comparación Visual

Comic Duplicate Scanner abre cada archivo CBZ, CBR, ZIP o RAR, extrae una muestra de páginas iniciales y genera un hash perceptual de cada una. Si suficientes hashes coinciden, el par se marca como posible duplicado — independientemente de lo que digan los nombres de archivo.

Controlas la sensibilidad de coincidencia. Un umbral más estricto requiere una coincidencia visual más cercana; uno más laxo capta más casos límite.
