## 为什么 CBZ 和 PDF 都很重要

CBZ（Comic Book ZIP）是数字漫画的首选格式，本质上是包含按序命名图片（通常为 JPEG 或 WebP）的 ZIP 压缩包，由漫画阅读器逐页显示。它体积紧凑、被专用应用广泛支持，且能很好地保留图片质量。

而 PDF 是通用文档格式，每台设备、每个操作系统、每个邮件客户端无需专用阅读器即可打开。PDF 可在 Kindle 上阅读、通过邮件发送、打印存档，并可在数百个不支持 CBZ 的应用中使用。

如果你管理着数字漫画库，迟早都需要跨越这道格式门槛——尤其是在共享文件或将内容导入 PDF 工作流时。

## BiblioFuse 如何在浏览器中完成 CBZ 转 PDF

[BiblioFuse 网页工具](https://bibliofuse.com/zh/webapp/)完全基于 WebAssembly（WASM）运行，转换逻辑在浏览器内执行，不向任何服务器发送数据。

转换过程如下：

1. 浏览器加载 WASM 模块——以接近原生的速度运行的编译沙箱二进制文件
2. 选择 CBZ 文件，载入浏览器内存
3. WASM 模块解压 ZIP 档案并读取图片序列
4. 图片按标准页面尺寸组装为 PDF
5. PDF 通过浏览器下载直接保存到设备——从不上传至任何位置

典型单期漫画（5–20 MB）的转换仅需数秒。大型卷册或高分辨率扫描可能需要 30–60 秒。

## 技术说明：CBZ 内部结构

CBZ 文件本质上是扩展名为 `.cbz` 的 ZIP 压缩包。将其重命名为 `.zip` 打开后，可看到按序命名的图片文件（如 `001.jpg`、`002.jpg`）。

BiblioFuse 转换器会：

- 读取 ZIP 索引以确定页面顺序
- 解压每张图片（JPEG、PNG、WebP 或 AVIF）
- 以原始分辨率将每张图片嵌入 PDF 页面
- 根据图片尺寸设置 PDF 页面大小
- 输出符合标准的 PDF 文件

由于图片以原始分辨率嵌入，输出的 PDF 与 CBZ 在视觉上完全一致，无任何质量损失。

## 常见问题

**转换器会保留我的文件副本吗？**  
不会。转换完全在浏览器内通过 WebAssembly 运行，文件从不离开你的设备。

**文件大小有限制吗？**  
没有服务器端限制，因为根本没有服务器参与。实际限制取决于浏览器可用内存，现代设备通常有数 GB。

**PDF 和 CBZ 看起来一样吗？**  
是的。图片以原始分辨率嵌入，唯一的区别是容器格式，视觉内容完全相同。
