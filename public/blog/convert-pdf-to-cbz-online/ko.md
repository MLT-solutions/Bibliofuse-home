# PDF를 CBZ로 온라인 변환하는 방법 (소프트웨어 불필요, 업로드 불필요)

스캔된 그래픽 노블을 PDF로 다운로드했다. Preview나 Acrobat에서는 잘 열리는데 코믹 리더에서는 뭔가 어색하다 — 페이지 로딩이 느리고, 줌이 흐릿해지고, 페이지 넘김이 즉각적이지 않다. CBZ가 만화에 더 적합하다는 걸 알지만, 또 다른 데스크탑 앱을 설치하거나 파일을 모르는 사이트에 올리고 싶지는 않다.

[BiblioFuse Web Tool](/ko/webapp/)은 이 문제를 브라우저에서 바로 해결한다. 계정 없음. 업로드 없음. 설치 없음. 파일은 내 기기를 떠나지 않는다.

## PDF가 만화에 맞지 않는 이유

PDF는 인쇄용 문서 레이아웃을 위해 설계됐다. 폰트, 벡터 그래픽, 텍스트 리플로우 처리에는 탁월하다. 하지만 만화는 다르다. 스캔된 만화는 페이지당 하나씩 이어지는 전면 이미지의 연속인데, PDF는 이 이미지들을 불필요한 내부 구조로 감싸 파일 크기만 키우고, 전용 이미지 뷰어보다 느린 렌더러를 쓰며, 만화 리더가 다음 페이지를 미리 캐시하는 것도 막는다.

CBZ는 번호가 붙은 이미지 파일의 시퀀스를 담은 ZIP 압축 파일이다. 오버헤드가 거의 없다. 만화 리더는 이미지를 미리 압축 해제해 메모리에 저장하고 페이지를 즉시 넘긴다. 줌 품질은 오직 이미지 파일에 달려 있다.

PDF 만화를 CBZ로 변환한다는 것은 PDF에서 페이지 이미지를 추출해 ZIP으로 재포장하는 일이다. 결과물은 만화 리더가 정확히 다루도록 만들어진 파일이 된다.

## BiblioFuse Web Tool이 하는 일

[BiblioFuse Web Tool](/ko/webapp/)은 브라우저 안에서 WebAssembly로 완전히 실행된다. WebAssembly는 샌드박스 환경에서 컴파일된 코드를 실행하므로, 서버에 아무것도 보내지 않고도 데스크탑 소프트웨어 수준의 파일 처리가 가능하다.

파일은 로컬 디스크에서 직접 브라우저 메모리로 읽힌다. 변환은 브라우저 탭에서 진행된다. 출력 파일은 디스크에 다시 저장된다. 어떤 시점에도 서버를 거치지 않는다.

라이선스 콘텐츠, 개인 아카이브, 혹은 남의 클라우드에 저장하고 싶지 않은 파일을 다룬다면 이 부분이 중요하다.

## PDF를 CBZ로 변환하는 방법

**1단계: 도구 열기**

[BiblioFuse Web Tool](/ko/webapp/)을 데스크탑 브라우저(Chrome, Firefox, Safari, Edge)에서 연다. 로그인이나 계정이 필요 없다. 무료 티어는 한 번에 최대 세 개 파일을 처리할 수 있다.

**2단계: PDF 파일 추가하기**

드롭 존을 클릭하거나 PDF 파일을 직접 드래그한다. 여러 파일을 한꺼번에 추가할 수 있다. 도구는 각 파일의 이름과 크기를 표시한다. 파일이 브라우저 가용 메모리보다 크면 처리 시작 전에 경고가 표시된다.

**3단계: CBZ 선택 후 변환**

변환 옵션에서 CBZ를 출력 형식으로 선택한다. 변환 버튼을 클릭한다. 도구는 각 파일을 순서대로 처리해 PDF의 페이지 이미지를 추출하고 CBZ 아카이브로 패키징한다. 진행 상황 표시기가 진행 정도를 알려준다. 변환이 끝나면 출력 파일이 다운로드 준비 상태로 나타난다.

다운로드된 파일은 원본과 같은 이름을 유지하며, `.pdf`가 `.cbz`로 바뀐다.

## 변환 중 이미지 품질

이 질문이 가장 자주 나오는데, 답은 원본 PDF가 어떻게 만들어졌느냐에 달려 있다.

**스캔 PDF:** 오래된 만화나 만화책 권호에서 가장 흔한 경우다. PDF 안에 스캔 이미지가 직접 저장되어 있다. 변환은 재인코딩 없이 이미지를 추출한다. CBZ 안의 이미지는 PDF의 이미지와 동일하다 — 화질 손실 없음, 추가 압축 없음.

**렌더링 PDF:** 일부 PDF는 벡터 아트나 텍스트 기반 레이아웃으로 생성된다. 이것을 CBZ로 변환하면 각 페이지가 설정된 해상도로 래스터화된다. [BiblioFuse Web Tool](/ko/webapp/)은 기본적으로 높은 해상도를 사용하므로 일반 읽기 줌 수준에서 선명하게 보인다.

**사전 압축 PDF:** 원본 PDF 생성 시 강한 JPEG 압축이 적용됐다면, 추출된 이미지에 그 압축이 그대로 반영된다. 도구는 압축을 추가하지 않지만, 이미 적용된 압축을 되돌릴 수도 없다.

스캔 만화의 경우 — 가장 일반적인 변환 시나리오 — 무손실 추출이 이루어진다. CBZ는 보통 PDF와 같은 크기이거나 약간 더 작으며, 화질 변화가 없다.

## 시리즈 일괄 변환

PDF 볼륨이 여러 권인 시리즈라면 [BiblioFuse Web Tool](/ko/webapp/)에서 한 세션에 전체 배치를 변환할 수 있다. PDF 파일을 한꺼번에 추가하고 CBZ로 목표를 설정한 다음 시작한다. 각 파일은 처리가 완료되는 대로 개별 다운로드가 가능하다.

무료 티어는 배치당 최대 세 개 파일을 처리한다. 더 긴 시리즈라면 같은 브라우저 세션에서 여러 배치를 실행하면 된다 — 배치 사이에 페이지를 새로 고칠 필요가 없다.

## 변환 후 CBZ 파일을 폰으로 옮기기

변환이 완료되고 CBZ 파일을 다운로드했다면, 폰으로 전송하는 깔끔한 방법들이 있다.

**Wi-Fi 전송 (가장 빠른 옵션):** iPhone에서 BiblioFuse를 사용한다면, 앱 내장 [Wi-Fi 전송 기능](/ko/)을 통해 같은 네트워크 상의 어느 컴퓨터에서든 브라우저 창에서 파일을 드래그할 수 있다. BiblioFuse를 열고 Wi-Fi 전송을 활성화한 다음, Mac이나 PC 브라우저에서 CBZ 파일을 라이브러리로 바로 드래그한다. USB 케이블, iTunes, 클라우드 스토리지 없이 가능하다.

**iCloud Drive:** BiblioFuse와 연결된 iCloud Drive 폴더에 파일을 넣으면 폰이 Wi-Fi에 연결될 때 자동으로 동기화된다.

**AirDrop:** Mac에서 Finder에 CBZ 파일을 선택하고 우클릭해 iPhone으로 AirDrop한다. 전송이 완료되면 "BiblioFuse에서 열기"를 탭한다.

## 자주 묻는 질문

**이미지와 텍스트 레이어가 모두 포함된 PDF도 처리되나요?**

그렇다. 스캔 이미지 레이어에 OCR 텍스트 오버레이가 있는 PDF도 올바르게 처리된다 — 도구는 시각적 페이지 이미지를 추출한다. 텍스트 레이어는 CBZ에 포함되지 않지만(CBZ는 이미지 아카이브 형식이다), 원본 PDF는 그대로 유지된다.

**파일 크기 제한이 있나요?**

도구가 강제하는 크기 제한은 없다. 실제 한도는 브라우저의 가용 메모리에 달려 있다 — 대부분의 데스크탑 브라우저는 수 기가바이트 파일도 문제없이 처리한다. 구형 하드웨어에서 매우 큰 파일은 느릴 수 있지만 보통 완료된다.

**모바일 브라우저에서도 작동하나요?**

작동하지만 데스크탑보다 성능이 눈에 띄게 느리다. 용량이 큰 PDF라면 데스크탑 브라우저를 권장한다.

**페이지 수 제한이 있나요?**

없다. PDF에 포함된 페이지 수에 관계없이 처리한다.

**CBZ에 들어가는 이미지 형식은 무엇인가요?**

원본 PDF에 저장된 이미지 형식을 그대로 사용한다. 스캔 만화의 경우 보통 JPEG다. 별도로 압축 단계를 활성화하지 않는 한 변환 중 트랜스코딩은 이루어지지 않는다.

## 전체 워크플로: PDF에서 폰까지

[BiblioFuse Web Tool](/ko/webapp/)은 소프트웨어 설치, 클라우드 계정, 파일 업로드 없이 PDF를 CBZ로 변환한다. 파일은 전체 과정 내내 내 기기에 머문다.

iPhone이나 iPad의 BiblioFuse와 함께 사용하면 군더더기 없는 완전한 워크플로가 완성된다: 브라우저에서 변환 → Wi-Fi로 폰에 전송 → 앱에서 읽기. 데스크탑 만화 관리자, 동기화 서비스, 중간 단계가 전혀 필요 없다.

변환하려고 했던 PDF 만화 폴더가 있다면, 지금 [BiblioFuse Web Tool](/ko/webapp/)을 열고 첫 번째 배치를 드롭해보자.
