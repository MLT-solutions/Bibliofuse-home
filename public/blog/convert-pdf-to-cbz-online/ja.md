# PDFをCBZにオンラインで変換する方法（ソフト不要・アップロード不要）

スキャンしたグラフィックノベルをPDFでダウンロードした。PreviewやAcrobatでは普通に開けるけれど、コミックリーダーで読むと何かがおかしい——ページの読み込みが遅い、ズームがぼやける、ページめくりがもたつく。CBZのほうがコミックに向いているとわかっていても、新たにデスクトップアプリをインストールしたくないし、ファイルを知らないサイトに送るのも嫌だ。

[BiblioFuse Web Tool](/ja/webapp/)は、これをブラウザだけで解決する。アカウント不要。アップロード不要。インストール不要。ファイルはデバイスの外に出ない。

## なぜPDFはコミックに向かないのか

PDFは印刷向けの文書レイアウトのために設計されており、フォント・ベクターグラフィック・テキストのリフローを得意とする。コミックには向いていない。スキャンコミックは1ページごとの画像の連続であり、PDFはそれらを余分な内部構造で包んでファイルサイズを増やし、専用ビューアより遅いレンダラーを使い、コミックリーダーによる先読みキャッシュを妨げる。

CBZは番号付き画像ファイルを収めたZIPアーカイブだ。オーバーヘッドがほとんどなく、リーダーは事前に画像を展開してメモリに保持し、ページをすぐにめくれる。PDFコミックをCBZに変換するとは、PDF内のページ画像を取り出してZIPに詰め直す作業にほかならない。

## BiblioFuse Web Toolの仕組み

[BiblioFuse Web Tool](/ja/webapp/)はWebAssemblyを使ってブラウザ内で完全に動作する。ファイルはローカルディスクからブラウザのメモリに直接読み込まれ、変換はブラウザのタブ内で実行され、出力ファイルはディスクに書き戻される。いかなる時点でもサーバーを経由しない。著作権素材やプライベートな蔵書を扱う場合でも安心だ。

## 変換手順

**ステップ1：ツールを開く**

[BiblioFuse Web Tool](/ja/webapp/)をデスクトップブラウザ（Chrome・Firefox・Safari・Edge）で開く。ログイン不要。無料プランは一度に3ファイルまで処理できる。

**ステップ2：PDFファイルを追加する**

ドロップゾーンをクリックするか、PDFを直接ドラッグする。複数ファイルを一度に追加可能。ブラウザのメモリを超えるサイズのファイルは処理前に警告が表示される。

**ステップ3：CBZを選んで変換する**

出力フォーマットにCBZを選び、「変換」をクリックする。各ファイルが順次処理され、完了したものからダウンロードできる。ファイル名は元のままで、拡張子だけ `.pdf` から `.cbz` に変わる。

## 画質について

- **スキャンPDF**：保存された画像をそのまま取り出すため再エンコードなし。CBZの画質はPDFと同一で、ファイルサイズもほぼ変わらない。
- **ベクター/レイアウトPDF**：各ページをラスタライズして出力。デフォルトで高解像度を使用するため、通常の閲覧ズームでは十分に鮮明。
- **圧縮済みPDF**：元のPDF作成時にJPEG圧縮が施されていれば、抽出画像にもその圧縮が残る。追加圧縮はされないが、既存の圧縮は取り除けない。

## シリーズの一括変換と端末への転送

複数巻をまとめて変換するには、全PDFを一度に追加してCBZを選べばよい。無料プランは1バッチ最大3ファイルで、同一セッション内でバッチを繰り返せるのでページ再読み込みは不要。

変換後のCBZをiPhoneに転送するには、BiblioFuseの[Wi-Fi転送](/ja/)機能が便利だ。同じネットワーク上のPCからブラウザでドラッグするだけで、USBもiTunesも不要。iCloud DriveやAirDropも使える。

## よくある質問

**OCRテキスト層が含まれるPDFは？** 正しく処理される。視覚的なページ画像を抽出し、元のPDFは変更されない。

**ファイルサイズの上限は？** 明示的な上限はなく、実際の制限はブラウザの利用可能メモリによる。デスクトップブラウザなら数GBのファイルも問題ない。

**モバイルブラウザでも使える？** 使えるが速度は遅い。大きなPDFにはデスクトップを推奨。

**CBZ内の画像フォーマットは？** 元のPDFに保存されているフォーマットのまま。スキャンコミックは通常JPEG。変換中のトランスコードは行われない。

[BiblioFuse Web Tool](/ja/webapp/)を開いて、最初のバッチをドロップしてみよう。
