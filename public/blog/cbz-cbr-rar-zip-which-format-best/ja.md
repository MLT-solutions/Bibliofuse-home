# CBZ・CBR・RAR・ZIP——マンガのアーカイブ形式はどれがベスト？

ダウンロードしたマンガの半分が `.cbz`、いくつかが `.cbr`、一つは普通の `.rar`。マンガリーダーで開けばすべて読めるけれど、ふと疑問が湧きます。形式によって違いはあるの？全部一つの形式に揃えるべき？そもそも何が違うの？

主要なマンガアーカイブ形式をわかりやすく整理し、[BiblioFuse](https://bibliofuse.com/ja/) がどれも問題なく読み込める理由を説明します。

## コミックアーカイブ形式の正体

CBZ・CBR・ZIP・RAR はどれもマンガ専用の形式ではありません。汎用のアーカイブコンテナで、中身は画像ファイル（主に JPEG・PNG、最近は WebP も増加）のフォルダだけです。マンガリーダーはアーカイブをメモリ上で展開し、画像を順番に表示します。

つまり形式の選択はほぼ完全に**圧縮アルゴリズム**と**互換性**の問題であり、マンガ向けの機能とは無関係です。

## CBZ——ほとんどのコレクションで最良の選択

CBZ は ZIP ファイルを `.cbz` に改名しただけです。以上。CBZ を展開すると画像フォルダが出てきます。

**CBZ がデフォルトとして最良な理由：**

- ZIP はオープンかつロイヤリティフリーで、すべての OS とアーカイブツールがネイティブ対応
- macOS・Windows・Linux は追加ソフト不要で ZIP を開ける
- 読み書き実装が容易なため、ほぼすべてのマンガリーダーが CBZ に対応
- RAR と異なり特許・ライセンスの問題なし
- アーカイブ全体を再圧縮せずページの追加・削除・並べ替えが可能

**圧縮効率のトレードオフ：** ZIP は Deflate 圧縮を使用し高速ですが最効率ではありません。すでに圧縮済みの JPEG 画像が入っているマンガアーカイブでは、RAR との差は通常 5% 未満です。PNG 中心のアーカイブでは RAR が 10〜20% 小さくなりますが、それよりも PNG 自体の体積が主な要因です。

**結論：** コレクションをゼロから作るなら CBZ を使いましょう。

## CBR——互換性は高いが技術的に劣る

CBR は RAR ファイルを `.cbr` に改名したものです。RAR は WinRAR の作者 Eugene Roshal が開発した独自形式で、ZIP よりやや高い圧縮率を実現します。特に PNG などの非圧縮画像形式で効果的です。

**CBR の問題点：**

- RAR は独自形式です。RAR ファイルを作成するには WinRAR か Unix 版 RAR のライセンス版が必要
- オープンソースの UnRAR ライブラリは読み取りのみで作成不可
- RAR 5 で新しいヘッダ形式が導入され、古いリーダーでは開けないことがある
- CBR の再パッキングや編集には追加ツールが必要

**CBR が多い理由：** WinRAR が Windows の主流アーカイバだった 2000 年代前半〜中頃のファイルが大半です。CBZ と比べて意味のある優位性がないため、新しいリリースで CBR が使われることはほぼありません。

**CBR を CBZ に変換すべきか？** ストレージが不足していて PNG 中心の CBR が多い場合は変換でスペースを節約できます。JPEG 中心のコレクションではサイズ差は無視できるほど小さく、変換の手間に見合いません。[BiblioFuse](https://bibliofuse.com/ja/) は CBR をネイティブで読み込めるため、読むためだけに変換する必要はありません。

## RAR と ZIP——基本形式

普通の `.rar` と `.zip` はマンガリーダーで開いた場合、CBR・CBZ と同じように動作します。違いはファイル拡張子だけ——マンガリーダーは内部形式ではなく拡張子を見て処理方法を決定します。一部のスキャンツールは `.cbz` や `.cbr` に改名せず元のアーカイブ拡張子のまま保存します。BiblioFuse は iOS で `.zip` と `.rar` を直接サポートしており、改名不要です。

## CB7 と CBT——マイナー形式

**CB7** は 7-Zip ファイル（`.7z`）を `.cb7` に改名したものです。7-Zip の LZMA2 圧縮はテキストなどの圧縮可能なデータで ZIP・RAR より大幅に優れますが、マンガはほぼ圧縮済みの JPEG 画像のため実際の節約は小さく、対応リーダーも少数です。

**CBT** は TAR ファイルを `.cbt` に改名したもので、圧縮は一切行わずファイルをまとめるだけです。Linux 由来のアーカイブに時折見られます。どちらも普及度は低いため、特別な理由がなければ CBZ を使い続けましょう。

## PDF——アーカイブではないが一般的

PDF はドキュメント形式であり画像コンテナではありません。PDF のマンガはページのレイアウト情報を含み、ページ単位の編集には専用ツールが必要です。出版社のデジタル販売作品やスキャン版の単行本でよく使われます。BiblioFuse は PDF を読み込めますが、画像ベース PDF の読書体験は同等の CBZ より劣ります——シーク速度が遅く、ページ単位のメタデータがなく、ファイルサイズも大きくなります。

マンガ PDF を CBZ に変換したい場合は、[BiblioFuse ウェブツール](https://bibliofuse.com/ja/webapp/) をご利用ください。ブラウザ上で完結し、アップロード不要です。

## 形式比較表

| 形式 | コンテナ | 圧縮 | 自由に作成 | 汎用対応 | 用途 |
|------|---------|------|-----------|---------|------|
| CBZ | ZIP | Deflate | ✅ 可 | ✅ 可 | 新規コレクション、自炊 |
| CBR | RAR | RAR/RAR5 | ⚠️ WinRAR 必要 | ✅ ほぼ全リーダー | 旧形式アーカイブ |
| ZIP | ZIP | Deflate | ✅ 可 | ✅ 可 | ZIP 直接対応リーダー |
| RAR | RAR | RAR/RAR5 | ⚠️ WinRAR 必要 | ✅ ほぼ全リーダー | 旧形式アーカイブ |
| CB7 | 7-Zip | LZMA2 | ✅ 可 | ⚠️ 一部のみ | 高圧縮実験 |
| CBT | TAR | なし | ✅ 可 | ⚠️ 一部のみ | Linux 由来アーカイブ |
| PDF | PDF | 各種 | ✅ 可 | ✅ 可 | 出版社 DRM フリー作品 |

## BiblioFuse でのすべての形式への対応

[BiblioFuse（iPhone・iPad 版）](https://bibliofuse.com/ja/) は CBZ・CBR・ZIP・RAR・CB7・CBT・EPUB・PDF をネイティブで読み込みます。インポート前にコレクションを変換する必要はありません。

すべての処理はデバイス上で行われ、サーバーへのアップロードは一切なし。大規模な多形式ライブラリでも、BiblioFuse がシリーズ別にファイルを整理し、iCloud 同期で各作品の読書進捗を追跡します。底層形式が CBZ でも CBR でも関係ありません。

BiblioFuse のツールタブでは iPhone 上で直接形式変換が可能です。CBR アーカイブを選択し、変換をタップして CBZ を選ぶと、アプリを離れずに再パッキングが完了します。

## 実践的な推奨事項

**ゼロから新しいコレクションを作る：** CBZ を使いましょう。オープンで汎用対応、取り扱いも簡単です。

**既存の CBR コレクション：** 特定の理由（ストレージ不足や CBR 非対応ツールなど）がなければ変換は不要です。CBR は BiblioFuse をはじめ現代のすべてのリーダーで正常に動作します。

**混在形式ライブラリ：** 見た目を揃えるためだけに統一しようとしないでください。BiblioFuse のように複数形式に対応するリーダーがあれば、単一形式コレクションを維持する必要はありません。

**出版社からの PDF マンガ：** 編集や複数巻の結合をしない限り PDF のままで問題ありません。必要な場合は BiblioFuse ウェブツールで CBZ に変換するのが合理的です。

形式の議論は、すべての形式を扱えるリーダーを持つことと比べれば、ずっと小さな問題です。それさえあれば、形式は気にならなくなります。
