# BiblioFuse가 iCloud를 통해 iPhone, iPad, Mac 간 독서 진행 상황을 동기화하는 방법

BiblioFuse는 iPhone, iPad, Mac 간에 독서 진행 상황, 태그, 별점, 읽기 방향을 iCloud를 통해 자동으로 동기화한다 — 수동 조작이 필요 없다. iOS 17 이상의 iCloud Drive에 저장된 EPUB, CBZ, CBR, PDF 파일에서 작동하며, 진행 상황을 파일 이름이 아닌 콘텐츠 해시에 연결해 파일을 이름 변경하거나 폴더를 옮겨도 정확히 유지된다. iCloud 동기화는 프리미엄 없이 포함된다.

출퇴근 중 iPhone으로 만화를 읽다가 — 80페이지까지 읽고 집에서 iPad로 전환하면 어디까지 읽었는지 페이지 썸네일을 뒤져야 한다. 또는 Mac에서 챕터를 읽고 다음 날 아침 iPhone을 들면 여전히 1페이지가 표시된다.

독서 진행 상황 동기화는 간단해 보이지만 대부분의 리더에서 미묘하게 실패한다. [BiblioFuse](/ko/)는 iCloud를 사용해 페이지 번호, 읽기 방향, 표시 설정을 포함한 정확한 읽기 위치를 iPhone, iPad, Mac 간에 수동 단계 없이 동기화한다.

## 진행 상황 동기화가 어려운 이유

단순한 방법 — 클라우드 문서에 페이지 번호 저장 — 은 여러 방면에서 실패한다: 다른 인코딩에서 페이지 수가 다른 볼륨을 처리하지 못하고, 읽기 방향을 동기화하지 않으며, 기기 간에 지속되는 "완료" 표시 메커니즘이 없다.

[BiblioFuse](/ko/)는 진행 데이터를 파일 이름이 아닌 파일 콘텐츠 해시에 연결된 메타데이터로 저장한다. CBZ를 이름 변경하거나 폴더 간에 이동해도 진행 데이터는 어느 기기에서나 올바르게 일치한다.

## iCloud 동기화 설정

**iPhone 또는 iPad**: **설정** → Apple ID → **iCloud** → **iCloud Drive**로 이동해 **BiblioFuse**를 찾고 토글이 켜져 있는지 확인한다.

**Mac**: **시스템 설정 → Apple ID → iCloud**를 열고 iCloud Drive를 활성화한 후 BiblioFuse가 목록에 있는지 확인한다.

이 설정이 켜지면 BiblioFuse가 모든 것을 자동으로 처리한다. "지금 동기화" 버튼은 없으며 기기가 온라인 상태일 때 변경 사항이 백그라운드에서 전파된다.

## 동기화되는 내용

**읽기 위치**: 정확한 페이지 번호. 87페이지에서 볼륨을 닫으면 다른 기기도 87페이지에서 열린다.

**읽기 방향**: 만화 볼륨을 오른쪽에서 왼쪽으로 설정하면 그 설정이 모든 기기의 볼륨을 따라간다.

**완료 상태**: 한 기기에서 볼륨을 읽음으로 표시하면 모든 곳에서 표시된다.

**태그와 평점**: 태그와 별 평점이 모든 기기에서 동기화된다.

## iCloud Drive 파일 동기화 작동 방식

[BiblioFuse](/ko/)는 라이브러리 파일 자체에도 iCloud Drive를 사용한다.

**Mac에서 iPhone으로**: Finder에서 **iCloud Drive → BiblioFuse**로 이동해 CBZ, CBR, EPUB 또는 PDF를 그 폴더에 넣으면 몇 분 내에 iPhone의 BiblioFuse에 나타난다.

Wi-Fi Transfer로 추가한 파일도 BiblioFuse의 iCloud 폴더에 저장되어 다른 기기에서 사용할 수 있다.

**대용량 전송에는 Wi-Fi Transfer**: 앱에서 **설정 → Wi-Fi 가져오기**를 켜고, 같은 Wi-Fi 네트워크의 브라우저에서 표시된 URL을 열고, 파일을 드래그한다. 로컬 네트워크 속도인 50–100 MB/s로 전송된다.

## Mac 홈 라이브러리: 파일 복사 없이 읽기

주요 컬렉션이 Mac에 있고 iPhone 저장소를 채우고 싶지 않다면 Mac 홈 라이브러리가 다른 접근 방식을 제공한다: 홈 Wi-Fi를 통해 Mac에서 iPhone으로 직접 볼륨을 스트리밍한다. 파일이 기기나 iCloud에 복사되지 않는다.

활성화: Mac의 BiblioFuse에서 **환경설정 → 홈 라이브러리**를 활성화하고, iPhone에서 BiblioFuse의 Mac 라이브러리 탭으로 전환한다. Mac 홈 라이브러리 세션의 독서 진행 상황도 iCloud를 통해 동기화된다.

## 오프라인 읽기

오프라인에서 읽으면 BiblioFuse가 진행 상황을 로컬에 저장하고 기기가 다음에 연결될 때 동기화한다. 두 기기가 충돌하면 페이지 번호가 높은 기기가 우선된다.

## 시작하기

[BiblioFuse](/ko/)는 iPhone과 iPad용 App Store에서 무료로 제공된다. iCloud 동기화는 프리미엄 없이 포함된다. 기기 설정에서 BiblioFuse의 iCloud Drive를 활성화하면 진행 상황이 즉시 동기화되기 시작한다.


## 자주 묻는 질문

**BiblioFuse는 iPhone과 iPad 간에 독서 진행 상황을 어떻게 동기화하나요?**
BiblioFuse는 진행 상황 메타데이터를 iCloud에 저장하고 파일 이름이 아닌 각 파일의 콘텐츠 해시에 연결합니다. 파일을 이름 변경하거나 이동해도 진행 상황이 정확히 동기화됩니다. 설정 → Apple ID → iCloud에서 BiblioFuse의 iCloud Drive를 활성화하면 동기화가 시작됩니다.

**BiblioFuse가 CBZ 및 만화 파일의 진행 상황도 동기화하나요?**
예. CBZ, CBR, EPUB, PDF에서 진행 상황 동기화가 작동합니다. CBZ/CBR은 정확한 페이지 번호가, EPUB는 챕터 내 정확한 위치가 동기화됩니다. 읽기 방향(왼쪽에서 오른쪽 또는 오른쪽에서 왼쪽)도 볼륨별로 동기화됩니다.

**BiblioFuse에서 iCloud 동기화가 작동하려면 무엇을 켜야 하나요?**
두 가지가 필요합니다. 설정 → Apple ID → iCloud에서 iCloud Drive가 활성화되어 있어야 하고, 그 목록에서 BiblioFuse가 켜져 있어야 합니다. 둘 다 켜면 동기화가 자동으로 시작됩니다. '지금 동기화' 버튼은 없습니다.

**오프라인에서 읽어도 iCloud 동기화가 작동하나요?**
예. 인터넷 없이 읽으면 BiblioFuse가 진행 상황을 로컬에 저장합니다. 기기가 다음에 iCloud에 연결될 때 업데이트를 전송합니다. 두 기기가 충돌하면 페이지 번호가 높은 기기가 우선됩니다.

**BiblioFuse의 iCloud 동기화와 Mac 홈 라이브러리의 차이는 무엇인가요?**
iCloud 동기화는 파일과 진행 상황 데이터를 클라우드에 복사해 모든 기기에서 사용할 수 있게 합니다. Mac 홈 라이브러리는 로컬 Wi-Fi를 통해 Mac에서 온디맨드로 스트리밍하며 기기나 iCloud에 아무것도 복사되지 않습니다. 어디서나 오프라인 접근을 위해서는 iCloud를, 저장 공간을 복제하지 않고 집에서 전체 컬렉션을 탐색하려면 Mac 홈 라이브러리를 사용하세요.
