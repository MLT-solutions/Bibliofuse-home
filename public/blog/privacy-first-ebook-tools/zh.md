当您将文件上传到云端电子书转换器或压缩工具时，实际上是在将数据交给第三方。隐私风险会迅速累积。

## WebAssembly 如何实现真正私密的文件处理？

WebAssembly（WASM）是一种二进制指令格式，能以接近原生应用的速度直接在浏览器中运行。它不是插件，不是服务器调用，也不是云端传输。以下是其实际意义：

1. **您的浏览器就是计算机。** 当您打开 [BiblioFuse 网页工具](/zh/webapp/) 时，所有处理逻辑——压缩算法、格式解析器、合并程序——作为 WASM 二进制文件一次性下载后在本地 CPU 上运行。
2. **文件从不经由网络传输。** JavaScript FileReader API 将您的 EPUB、CBZ 或 PDF 直接从磁盘加载到浏览器内存中。没有任何数据被序列化到请求体中发送到任何地方。
3. **不涉及任何服务器基础设施。** 没有后端服务器在等待接收您的上传、处理文件并返回结果。整个处理流程在浏览器沙箱中执行。
4. **结果直接从内存中下载。** 处理完成后，浏览器创建一个指向内存中输出数据的本地对象 URL。您点击下载，文件保存到设备——再次全程不接触任何服务器。

这种架构不是隐私政策中的承诺，而是由 Web 平台工作原理所强制保障的技术承诺。
