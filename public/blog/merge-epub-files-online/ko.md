# 여러 EPUB 파일을 하나로 합치는 방법 (온라인, 개인 정보 보호, 무료)

BiblioFuse 웹 도구는 여러 EPUB 파일을 브라우저 안에서 완전히 처리해 단 하나의 전자책으로 합쳐줍니다. 소프트웨어 설치가 필요 없고, 계정을 만들 필요도 없으며, 파일이 어떤 서버에도 업로드되지 않습니다. EPUB 파일을 불러와서 순서대로 정렬한 뒤 몇 초 만에 합쳐진 결과를 다운로드하세요. 무료 플랜은 배치당 최대 3개 파일을 처리합니다. BiblioFuse는 EPUB, CBZ, CBR, PDF, ZIP 등 다양한 형식을 지원하며 WebAssembly 기반으로 모든 처리가 로컬 기기에서 이루어집니다.

## EPUB 파일을 온라인으로 합치고 싶은 상황

판타지 소설 전 8권을 다운로드했는데, 각 권이 별도의 EPUB 파일로 되어 있다고 상상해 보세요. 한 권을 읽고 나면 매번 라이브러리 화면으로 돌아가 다음 파일을 수동으로 열어야 합니다. 또는 화(話)별로 배포된 만화를 갖고 있어서 끊김 없이 전체 스토리를 읽고 싶은 경우도 있을 것입니다. 콘텐츠를 여러 파일로 나누는 것은 출판사 입장에서는 합리적인 결정이지만, 연속적으로 읽고 싶은 독자에게는 불편함을 줍니다.

가장 자연스러운 해결책은 모든 파일을 리더에 불러오기 전에 하나의 EPUB으로 합치는 것입니다. 문제는 이 작업을 제대로 수행하는 것이 보기보다 훨씬 복잡하다는 점입니다.

## EPUB 파일 합치기가 생각보다 어려운 이유

EPUB 파일은 단순한 텍스트 묶음이 아닙니다. 내부적으로는 ZIP 아카이브로, 읽기 순서를 정의하는 XML 형식의 spine, 모든 리소스를 나열하는 manifest, XHTML 형식의 챕터 파일, `toc.ncx` 또는 `nav.xhtml` 형식의 목차, 그리고 폰트와 이미지 같은 내장 리소스로 구성되어 있습니다. 두 EPUB을 합칠 때는 이 모든 구조를 조화롭게 통합해야 합니다. 챕터 ID 중복 제거, spine 항목 재정렬, 목차 항목 재번호 부여, 충돌하는 메타데이터 해결 등이 필요합니다.

EPUB를 합칠 수 있다고 주장하는 대부분의 온라인 도구는 ZIP 바이트를 단순히 이어붙이거나(손상된 파일 생성) 목차를 통째로 삭제해 버립니다. BiblioFuse는 완전한 구조적 병합을 수행합니다. 챕터가 순서에 따라 재번호 부여되고, 목차가 모든 소스 파일로부터 새로 구축되며, 출력 파일은 EPUB 유효성 검사를 통과합니다.

## BiblioFuse로 EPUB 파일을 온라인으로 합치는 방법

전체 과정이 1분 이내에 완료됩니다.

**1단계: BiblioFuse 웹 도구 열기**

[BiblioFuse 웹 도구](/ko/webapp/)로 이동합니다. 이 도구는 WebAssembly를 사용해 완전히 브라우저에서 실행됩니다. 어느 시점에도 서버로 데이터가 전송되지 않습니다.

**2단계: 읽기 순서에 맞게 EPUB 파일 추가하기**

파일 선택기를 클릭하거나 EPUB 파일을 드롭 영역에 드래그합니다. EPUB, CBZ, CBR, PDF, ZIP 형식을 지원합니다. 합쳐진 결과물에서 표시되길 원하는 순서대로 파일을 추가하세요. 1권을 먼저, 그다음 2권, 이런 식으로 추가합니다.

**3단계: 확인 후 병합 실행**

파일 목록이 올바른 순서로 나열되어 있는지 확인합니다. 필요하면 드래그해서 순서를 바꿀 수 있습니다. 병합 작업을 선택하고 실행을 확인합니다.

**4단계: 합쳐진 EPUB 다운로드**

병합 처리는 완전히 브라우저 내에서 이루어집니다. 완료되면 다운로드 버튼을 클릭합니다. 출력 파일은 Apple Books, Kobo, [BiblioFuse](/ko/)를 포함한 모든 표준 리더에서 열 수 있는 유효한 EPUB입니다.

무료 플랜으로는 배치당 최대 3개 파일을 합칠 수 있습니다. 더 긴 시리즈의 경우 여러 번에 나눠 처리한 후 결과물을 다시 합치는 방식을 사용할 수 있습니다.

## EPUB 파일 합치기의 실용적인 활용 사례

**판타지·SF 다권 시리즈.** 출판사는 독자가 전체 시리즈를 연속으로 읽으려는 경우에도 각 권을 별도의 EPUB 파일로 출시하는 것이 일반적입니다. 합치면 모든 권의 목차가 통합된 하나의 파일이 생기고, 모든 권의 어느 챕터로도 바로 이동할 수 있습니다.

**만화 합본판.** 많은 만화 작품이 화별 또는 단행본 단위로 별도의 EPUB으로 배포됩니다. 이를 합본 스타일의 단일 파일로 합치는 것은 라이브러리를 간결하게 유지하고 싶은 독자들 사이에서 흔히 쓰는 방법입니다.

**분권 출판된 학술 교재.** 일부 대학 교재는 1부, 2부, 3부 등으로 분리되어 출판됩니다. 하나의 파일로 합치면 주석 추가와 교차 참조가 훨씬 편해지며, 특히 시험 준비에 유용합니다.

**연재 소설.** Royal Road 같은 플랫폼에서 내보낸 웹 소설은 여러 EPUB 파일로 나뉘어 있는 경우가 많습니다. 합치면 라이브러리가 정돈되고 읽기 진행 상황을 한 곳에서 관리할 수 있습니다.

## 자주 묻는 질문

### EPUB 파일을 합치면 목차가 유지되나요?

네. BiblioFuse는 모든 소스 EPUB 파일로부터 완전한 목차를 재구축합니다. 각 권의 챕터는 합쳐진 목차에서 연속된 블록으로 표시되고, 원본 챕터 제목은 그대로 보존됩니다. EPUB 2(`toc.ncx`)와 EPUB 3(`nav.xhtml`) 모두 지원하는 모든 리더에서 탐색 기능이 정상 동작합니다.

### 작가나 메타데이터가 다른 EPUB 파일을 합칠 수 있나요?

가능합니다. 소스 파일의 메타데이터가 다를 경우(작가, 출판사, 표지 이미지 등) BiblioFuse는 목록의 첫 번째 파일 메타데이터를 출력 파일의 기본값으로 사용합니다. 병합 전 설정 패널에서 제목, 작가 등을 재정의하거나 병합 후 EPUB 편집기로 수동 편집할 수도 있습니다.

### EPUB 파일을 온라인으로 합치는 것이 안전한가요? 파일이 서버에 업로드되나요?

파일은 절대 업로드되지 않습니다. [BiblioFuse 웹 도구](/ko/webapp/)는 WebAssembly를 통해 브라우저 내에서 100% 실행됩니다. 파일은 처음부터 끝까지 사용자의 기기에만 머뭅니다. 서버도, 계정도, 클라우드 스토리지도 전혀 관여하지 않습니다. 비행기 모드를 켠 상태에서도 도구가 정상 작동하는 것으로 확인할 수 있습니다.

### 합친 후 책갈피와 하이라이트는 어떻게 되나요?

책갈피와 하이라이트는 EPUB 파일 자체가 아니라 읽기 앱에 저장됩니다. 파일을 합쳐서 새로운 합본 EPUB를 불러오면 앱이 새 파일로 인식하기 때문에 개별 권에서의 기존 읽기 위치는 이전되지 않습니다. 합본 파일의 처음부터, 또는 새로운 통합 목차에서 챕터를 찾아 독서를 시작하세요.

### 3개 이상의 EPUB 파일을 한 번에 합칠 수 있나요?

무료 플랜은 배치당 최대 3개 파일을 지원합니다. 더 긴 시리즈는 단계별로 처리할 수 있습니다. 1~3권을 A 파트로, 4~6권을 B 파트로 합친 뒤 최종적으로 각 파트를 합치는 방식입니다. 한 번에 더 많은 파일을 처리하려면 계정 업그레이드도 가능합니다.

### 합쳐진 EPUB가 Kindle과 Kobo에서 작동하나요?

네, 다만 Kindle은 한 단계가 추가됩니다. Kobo 기기는 EPUB를 기본으로 지원하므로 합쳐진 EPUB를 바로 열 수 있으며 통합 목차도 정상적으로 표시됩니다. Kindle은 고유한 형식(MOBI, AZW3)을 사용하므로 Calibre나 아마존의 Kindle 전송 도구로 변환한 후 사이드로드해야 합니다. BiblioFuse가 출력하는 합본 EPUB는 규격을 준수하여 변환이 원활합니다.
