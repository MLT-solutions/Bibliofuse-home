# BiblioFuse 라이브러리 관리: 폴더, 전송, Mac 스트리밍

[BiblioFuse](https://bibliofuse.com)는 간단한 원칙 위에 만들어졌다. 파일은 당신이 놓은 곳에 그대로 있다. 앱은 파일을 이동하거나, 이름을 바꾸거나, 독자적인 형식에 가두지 않는다. 책이 어디에 있는지, 그리고 어떻게 그곳에 가져다 놓는지를 이해하는 것이 당신을 위해 작동하는 라이브러리와 끊임없이 씨름해야 하는 라이브러리의 차이를 만든다.

## 책 추가 방법: 네 가지 경로

### Mac과 Android — 폴더 추가

Mac과 Android에서 BiblioFuse는 파일을 소유하는 것이 아니라 폴더를 감시한다. 사이드바에서 **폴더 추가**를 탭하고 아무 위치나 지정하면 된다 — 다운로드 폴더, 외장 드라이브, NAS 어디든 괜찮다. BiblioFuse가 내용을 인덱싱하고 변경 사항을 추적하며, 파일 자체는 건드리지 않는다.

### iOS — BiblioFuse 폴더에 복사하기

iPhone과 iPad는 샌드박스 파일 모델을 사용하기 때문에, 앱이 볼 수 있는 위치에 책을 복사해야 한다. 두 가지 위치가 작동한다.

**iCloud Drive → BiblioFuse** — 파일 앱을 열고 iCloud Drive → BiblioFuse로 이동한다. 전자책 폴더를 여기에 복사하거나 이동하면, 각 폴더가 BiblioFuse 목록에 별도 항목으로 나타난다. iCloud Drive → BiblioFuse 루트(하위 폴더가 아닌 곳)에 직접 넣은 책이나 파일은 앱에서 **iCloud**라는 폴더 아래에 표시된다. 여기에 저장된 책은 독서 진행 상황이 자동으로 동기화된다. iPhone, iPad, Mac 간에 읽던 위치가 동기화되어 어느 기기에서든 이어서 읽을 수 있다.

**내 iPhone → BiblioFuse** — 기기 전용 로컬 저장을 원하면 전자책 폴더를 여기에 복사한다. 폴더는 목록에 동일하게 나타나지만, 진행 상황은 해당 기기에만 남는다.

두 경우 모두 폴더 단위로 넣을 것 — 개별 파일이 아니라. 만화 시리즈의 여러 권이 담긴 폴더는 목록에서 하나의 항목이 되고, 모든 권이 그 안에 들어간다.

### Wi-Fi 전송

내장 브라우저 기반 가져오기 서버를 사용하면 케이블이나 동기화 소프트웨어 없이 어떤 컴퓨터에서든 파일을 가져올 수 있다.

- **iOS**: **설정 → Wi-Fi 가져오기**로 이동해 활성화한다. 로컬 URL이 표시되면 데스크톱 브라우저에서 열고 파일을 드래그해 넣는다. 파일은 **내 iPhone → BiblioFuse → import**에 저장되고 라이브러리에 자동으로 나타난다.
- **Android**: Wi-Fi 전송 설정 시 대상 폴더를 선택한다.

### 도구 탭에서 처리된 책

[도구 탭](/ko/webapp)을 사용해 파일을 압축하거나 합칠 때, 결과물은 다음 위치로 간다.

- **iOS**: **내 iPhone → BiblioFuse → export** — 처리된 파일이 자동으로 여기에 나타난다.
- **Android**: 도구 실행 시 출력 폴더를 선택한다.

[BiblioFuse 웹 도구](/ko/webapp)는 어떤 브라우저에서든 동일한 기능을 제공한다 — Windows나 Linux에서 특히 유용하다. 압축으로 절약할 수 있는 저장 공간에 대한 자세한 내용은 [만화와 전자책 파일 크기 가이드](/ko/blog/reduce-comic-ebook-file-size)를 참조하자.

### Mac: 오른쪽 클릭 한 번으로 iCloud에 전송

Mac에서 읽다가 iPhone에서도 보고 싶은 책이 있으면, 책 표지를 오른쪽 클릭하고 **iCloud로 보내기**를 선택한다. BiblioFuse가 파일을 **iCloud Drive / BiblioFuse**에 직접 복사한다 — iPhone이 감시하는 동일한 컨테이너다. 몇 분 안에 아무 수동 작업 없이 전화기에 나타난다.

여러 책을 한 번에 보내려면 선택 모드(오른쪽 상단 체크 아이콘)를 켜고 원하는 책을 선택한 다음, 액션 바에서 **iCloud로 보내기**를 탭한다.

## 라이브러리 그리드: 길게 누르기 옵션

iOS에서 책 표지를 길게 누르면 컨텍스트 메뉴가 열린다.

- **열기** — 바로 읽기로 이동
- **공유** — 다른 앱에 파일 전달
- **이름 변경** — 표시 이름 변경 (디스크의 파일명은 바뀌지 않음)
- **메타데이터 편집** — 태그 추가 및 1~5점 별점 설정
- **제거** — 두 가지 옵션이 있는 확인 창이 나타남:
  - *라이브러리에서 제거* — 라이브러리 항목을 삭제하되 파일은 디스크에 그대로 남김
  - *저장소에서 삭제* — 항목과 실제 파일 모두 삭제

도구 탭으로 책을 보내려면, 선택 모드(첫 번째 책을 길게 누르고 나머지를 탭해서 추가)에 들어간다. 하단에 **도구** 버튼(마법 지팡이 아이콘)이 있는 툴바가 나타난다. 탭하면 선택한 책들이 압축이나 합치기 대기열에 들어간다.

## Mac 라이브러리를 iPhone으로 스트리밍

메인 컬렉션이 Mac에 있다면, 아무것도 복사할 필요가 없다. [BiblioFuse](/ko/)는 Mac에서 iPhone으로 직접 스트리밍할 수 있다 — 단, 네트워크가 바뀔 때 불안정해질 수 있는 기본 Wi-Fi 방식 대신 Tailscale을 사용하자.

Tailscale은 기기 사이에 비공개 암호화 네트워크를 만든다. 설정 후에는 모바일 데이터, 공공 Wi-Fi, 집 어디에서든 iPhone이 Mac에 접근할 수 있다.

### 1단계: 두 기기에 Tailscale 설치

1. Mac: [tailscale.com/download](https://tailscale.com/download) 또는 Mac App Store에서 다운로드.
2. iPhone: App Store에서 "Tailscale" 검색.
3. 두 기기에 **같은 Tailscale 계정**으로 로그인.

연결되면 두 기기가 비공개 네트워크 안에서 서로 통신할 수 있다.

### 2단계: Mac에서 스트리밍 활성화

Mac용 BiblioFuse에서 왼쪽 사이드바를 아래로 스크롤해 **스트리밍** 섹션을 찾고 켠다. 두 개의 엔드포인트가 나타난다: LAN/Wi-Fi용과 **원격 / Tailscale**용. Tailscale이 안정적인 경로다.

### 3단계: iPhone 권한 확인

두 가지 iOS 권한이 필요하다.

1. **로컬 네트워크** — **설정 → 개인 정보 보호 및 보안 → 로컬 네트워크**로 이동해 BiblioFuse를 켠다.
2. **iCloud** — **설정 → [내 이름] → iCloud**로 이동해 BiblioFuse를 켠다.

빠른 확인: iPhone의 BiblioFuse에서 **설정 → 권한**으로 이동한다. 초록 체크 표시는 정상, 빨간 표시를 탭하면 수정이 필요한 시스템 설정으로 바로 이동한다.

### 4단계: iPhone에서 연결

1. BiblioFuse의 **라이브러리** 탭을 연다.
2. 사이드바(iPad에서는 왼쪽 패널, iPhone에서는 사이드바 아이콘 탭)를 맨 아래까지 스크롤한다 — **Mac** 섹션에 발견된 스트리밍 엔드포인트 목록이 있다.
3. 이름에 **(iCloud)**가 포함된 항목을 찾는다. 예: *홍길동의 MacBook Pro (iCloud)*.
4. 탭한다. 보안 연결이 설정되기까지 10~30초가 걸린다.

Mac 라이브러리 폴더가 사이드바에 로컬 저장소처럼 나타난다. 평소처럼 탐색하고 읽으면 된다.

### 스트리밍되는 것 vs. 다운로드되는 것

| 형식 | 동작 방식 |
|------|---------|
| CBZ / CBR | 페이지별 스트리밍 — 다운로드 불필요, 즉시 열림 |
| EPUB / TXT | 기기에 먼저 다운로드된 후 열림 |

**독서 진행 상황 안내:** 스트리밍 세션의 진행 상황은 Mac의 로컬 데이터베이스에 저장되며, iCloud에는 저장되지 않는다. 다른 기기와 동기화되지 않고 Mac에 남는다.

## 빠른 팁

- **iCloud가 가장 쉬운 동기화 방법**: iCloud Drive → BiblioFuse에 있는 책은 모든 Apple 기기에서 진행 상황 추적과 함께 자동으로 이용할 수 있다.
- **export 폴더는 임시 보관 공간**: iOS에서 압축 후 내 iPhone → BiblioFuse → export에서 결과물을 찾아 장기 보관할 위치로 이동하자.
- **스트리밍이 안 되면 먼저 권한 확인**: 대부분의 연결 문제는 로컬 네트워크 또는 iCloud가 꺼져 있어서 발생한다.
