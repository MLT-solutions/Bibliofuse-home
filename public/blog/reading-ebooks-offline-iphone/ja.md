# iPhoneで完全オフライン電子書籍を読む：クラウド不要

14時間のフライトに乗り込もうとしています。機内持ち込み荷物にはiPhoneが入っており、電子書籍300冊、漫画40巻、グラフィックノベル数冊が入っています。航空会社のWi-Fiは有料で低速。[BiblioFuse](https://bibliofuse.com/ja/)を開くと、すべての本が瞬時に開き、前回読んでいたページが表示され、ネットワークへのリクエストは一切発生しません。

これは特別な設定ではなく、デフォルトの動作です。BiblioFuseはすべてをデバイスにローカル保存し、接続可能な時にバックグラウンドでiCloudを通じて進捗を同期します。読書にクラウドアカウントは不要です。ストリーミングなし、読み込み画面なし。

## クラウド依存が問題な理由

多くの読書アプリはサブスクリプションを念頭に設計されています。クラウドは機能ではなく、囲い込みの仕組みです。本はサーバーに保存され、進捗もサーバー側にあり、会社が閉鎖したり利用規約を変更したりサブスクリプションが切れると、アプリは機能しなくなります。

Apple Booksでさえクラウドファーストのモデルを採用しています。Appleで「購入」した本はDRMとiCloudダウンロードで守られており、明示的にダウンロードするまで真の意味でローカルではなく、Apple IDに紐付けられています。

大量のDRMフリーライブラリを持つユーザー—CBZコミックアーカイブ、出版社のEPUBファイル、PDFの教科書—にとって、このモデルは適していません。ファイルはあなたのものです。サービスに預けることなく読みたいはずです。

## BiblioFuseがライブラリをローカルに保存する仕組み

BiblioFuseはオフラインファーストで設計されています。Wi-Fi転送、AirDrop、ファイルアプリなどの方法でアプリにファイルを追加すると、iPhoneのアプリのローカルサンドボックスにコピーされます。Macなどにあるオリジナルファイルはそのまままのまま残り、BiblioFuseはデバイス上に独自のコピーを持ちます。

つまり：

- ネットワーク状態に関わらず、ライブラリ全体にいつでもアクセス可能
- 読書進捗はローカルで記録され、接続時にiCloudに同期
- アプリの使用にアカウントは不要—Apple IDさえ必要ありません（iCloud同期を有効にする場合は必要）
- 飛行機内、地下鉄トンネル内、電波のない農村部でもまったく同じように動作

BiblioFuseはEPUB、CBZ、CBR、ZIP、PDFおよび通常の画像アーカイブに対応しています。すべてのフォーマットがオフラインで動作し、挙動の違いはありません。

## iPhoneに電子書籍やコミックを読み込む方法

クラウドサービスに依存せずにBiblioFuseにファイルを読み込む方法は3つあります。

### 方法1：Wi-Fi転送（ローカルネットワーク、インターネット不要）

Wi-Fi転送は、ローカルWi-Fiネットワーク経由でMacまたはPCからiPhoneにファイルを転送するBiblioFuseの組み込みツールです。インターネットは使用しません。

1. iPhoneで[BiblioFuse](https://bibliofuse.com/ja/)を開き、下部の**ツール**をタップします。
2. **Wi-Fi転送**をタップし、表示されるIPアドレスをメモします。
3. MacでブラウザーをゾしてそのIPアドレスにアクセスします。
4. EPUB、CBZ、またはPDFファイルをブラウザーのアップロードエリアにドラッグします。

ファイルはローカルネットワーク速度—通常30〜80 MB/s—で転送され、ライブラリに直接追加されます。中間アプリなし、iCloudなし、インターネット接続なし。

### 方法2：AirDrop

他のAppleデバイスから素早く数ファイルを転送する場合：

1. Mac、iPad、または別のiPhoneからAirDropでファイルを共有します。
2. iPhoneで転送を承認します。
3. **ファイル → ダウンロード**でファイルを見つけます。
4. ファイルを選択し、共有をタップしてBiblioFuseを選択します。

AirDropはピアツーピアWi-Fiを使用します。モバイルデータやインターネットアクセスの有無に関わらず、自分のAppleデバイス間で動作します。

### 方法3：USB経由のファイルアプリ

USBでiPhoneをMacに接続し、Finder（macOS Catalina以降）を使用：

1. FinderでiPhoneを選択します。
2. **ファイル**をクリックします。
3. BiblioFuseのファイルエリアにファイルを直接ドラッグします。

USB転送は大きなライブラリに最も速く信頼性の高い方法です—数百巻を数分で転送できます。

## オフライン中の読書進捗はどうなるか

BiblioFuseはデバイス上のローカルデータベースに読書進捗を書き込みます。すべてのページめくり、ブックマーク、注釈は最初にローカルに保存されます。インターネット接続が利用可能になると、変更は自動的にiCloudに同期されます。

iPhoneでオフライン読書後、iPadやMacでBiblioFuseを開くと進捗が反映されています。同期は最新のタイムスタンプを使用するため、両方のデバイスが同時にオフラインであってもコンフリクトやデータ損失はありません。

同期を起動するために何かする必要はありません。デバイスが接続されると、BiblioFuseがバックグラウンドで自動的に処理します。

## オフライン対応フォーマット

BiblioFuseは以下のフォーマットを完全オフラインで読み込みます：

| フォーマット | 主な用途 |
|------------|---------|
| EPUB | 電子書籍、小説、ノンフィクション |
| CBZ | コミックアーカイブ（ZIPベース） |
| CBR | コミックアーカイブ（RARベース） |
| ZIP | 画像アーカイブ |
| PDF | 教科書、アートブック、スキャン文書 |

すべてのフォーマットがオンラインと同じ速度でオフラインでも開きます。制限モード、透かし、広告はありません。

## よくある質問

**BiblioFuseをオフラインで使うにはAppleアカウントが必要ですか？**

いいえ。BiblioFuseはApple IDなしで完全に動作します。iCloudを使用しない場合、進捗はローカルに保存され他のデバイスと同期されませんが、読書、ブックマーク、ライブラリ管理など他の機能はすべて正常に動作します。

**Apple以外で購入したDRMフリーの電子書籍を読めますか？**

はい。BiblioFuseは入手先を問わずDRMフリーのEPUBファイルを読み込みます。出版社、Project Gutenberg、またはDRMフリーEPUBダウンロードを提供するストアからの電子書籍はすべて動作します。DRM保護されたファイルは開けません。

**ファイルはデバイスのストレージを使いますか？**

はい。ファイルはローカルに保存されるためデバイスのストレージを使用します。一般的なEPUL小説は1〜5 MB、高品質なCBZ漫画は80〜200 MBです。数十〜数百冊程度であれば、現代のiPhoneでストレージが問題になることはほとんどありません。

**機内モードでもアプリは使えますか？**

完全に使えます。機内モードをオンにしてBiblioFuseを開くと、ライブラリ全体がそこにあり、進捗追跡もブックマークも通常通り動作します。iCloud同期は再接続するまで一時停止されるだけです。

## まとめ

BiblioFuseは「読書生活がサーバーの可用性に依存すべきではない」という前提で設計されています。ファイルはローカルにあり、進捗はまずローカルに記録されます。光ファイバー接続上にいても1万メートルの上空にいても、読書体験はまったく同じです。

電子書籍や漫画のライブラリをお持ちで、クラウドに依存せずiPhoneで読みたい方には、[BiblioFuse](https://bibliofuse.com/ja/)が直接の解決策です。Wi-Fi、AirDrop、またはUSBで一度ファイルを読み込めば、どこでも読めます。
