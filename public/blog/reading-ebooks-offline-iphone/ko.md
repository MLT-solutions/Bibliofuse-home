# iPhone에서 완전히 오프라인으로 전자책 읽기: 클라우드 불필요

14시간 비행기에 탑승하려 합니다. 기내 수하물에는 전자책 300권, 만화 40권, 그래픽 노블 여러 권이 담긴 iPhone이 있습니다. 항공사 Wi-Fi는 유료에 느립니다. [BiblioFuse](https://bibliofuse.com/ko/)를 열면 — 모든 책이 마지막으로 읽던 페이지에서 즉시 열리며, 네트워크 요청은 전혀 없습니다.

이것은 특별한 설정이 아닙니다. 이것이 기본 동작입니다. BiblioFuse는 모든 내용을 기기에 로컬로 저장하고, 연결이 가능할 때 iCloud를 통해 백그라운드에서 진행 상황을 동기화합니다. 읽기 위해 클라우드 계정이 필요하지 않습니다. 스트리밍 없음. 로딩 화면 없음.

## 클라우드 의존이 문제인 이유

대부분의 독서 앱은 구독을 염두에 두고 설계되었습니다. 클라우드는 기능이 아니라 잠금 메커니즘입니다. 책은 서버에 있고, 진행 상황도 서버에 있으며, 회사가 문을 닫거나 약관을 변경하거나 구독이 만료되면 앱은 작동을 멈춥니다.

Apple Books조차 클라우드 우선 모델을 사용합니다. Apple을 통해 "소유한" 책은 DRM과 iCloud 다운로드로 보호되며, 명시적으로 다운로드할 때까지 진정한 의미의 로컬이 아니고, Apple ID에 연결되어 있습니다.

CBZ 만화 아카이브, 출판사의 EPUB 파일, PDF 교재 등 대규모 DRM 무료 라이브러리를 가진 사람들에게는 이 모델이 맞지 않습니다. 파일은 여러분의 것입니다. 서비스에 넘기지 않고 읽고 싶을 것입니다.

## BiblioFuse가 라이브러리를 로컬에 저장하는 방법

BiblioFuse는 처음부터 오프라인 우선으로 설계되었습니다. Wi-Fi 전송, AirDrop, 파일 앱 또는 다른 방법으로 앱에 파일을 추가하면, iPhone의 앱 로컬 샌드박스에 복사됩니다. Mac이나 다른 곳의 원본 파일은 그대로 남아 있으며, BiblioFuse는 기기에 자체 복사본을 보유합니다.

이는 다음을 의미합니다:

- 네트워크 상태에 관계없이 전체 라이브러리에 언제든지 접근 가능
- 읽기 진행 상황이 로컬에 기록되고 연결 시 iCloud에 동기화됨
- 앱 사용에 계정이 필요하지 않음 — Apple ID도 필요 없음 (iCloud 동기화를 활성화하면 필요)
- 비행기, 지하철 터널, 신호 없는 농촌 지역에서도 동일하게 작동

BiblioFuse는 EPUB, CBZ, CBR, ZIP, PDF 및 일반 이미지 아카이브를 지원합니다. 모든 형식이 오프라인에서도 동작하며 동작 차이가 없습니다.

## iPhone에 전자책과 만화를 로드하는 방법

클라우드 서비스에 의존하지 않고 BiblioFuse에 파일을 로드하는 세 가지 방법이 있습니다.

### 방법 1: Wi-Fi 전송 (로컬 네트워크, 인터넷 불필요)

Wi-Fi 전송은 BiblioFuse의 내장 도구로, 로컬 Wi-Fi 네트워크를 통해 Mac이나 PC에서 iPhone으로 파일을 전송합니다. 인터넷을 사용하지 않습니다.

1. iPhone에서 [BiblioFuse](https://bibliofuse.com/ko/)를 열고 하단의 **도구**를 탭합니다.
2. **Wi-Fi 전송**을 탭하고 표시된 IP 주소를 기록합니다.
3. Mac에서 브라우저를 열고 해당 IP 주소로 이동합니다.
4. EPUB, CBZ 또는 PDF 파일을 브라우저 업로드 영역으로 드래그합니다.

파일은 로컬 네트워크 속도 — 일반적으로 30~80 MB/s — 로 전송되어 라이브러리에 직접 들어옵니다. 중간 앱 없음, iCloud 없음, 인터넷 연결 사용 없음.

### 방법 2: AirDrop

다른 Apple 기기에서 몇 개의 파일을 빠르게 이동할 때:

1. Mac, iPad 또는 다른 iPhone에서 AirDrop을 통해 파일을 공유합니다.
2. iPhone에서 전송을 수락합니다.
3. **파일 → 다운로드**에서 파일을 찾습니다.
4. 파일을 선택하고 공유를 탭한 다음 BiblioFuse를 선택합니다.

AirDrop은 피어투피어 Wi-Fi를 사용합니다. 기기에 모바일 데이터나 인터넷 접속이 있는지 여부에 관계없이 자신의 Apple 기기 간에 작동합니다.

### 방법 3: USB를 통한 파일 앱

USB로 iPhone을 Mac에 연결하고 Finder(macOS Catalina 이상)를 사용:

1. Finder에서 iPhone을 선택합니다.
2. **파일**을 클릭합니다.
3. BiblioFuse의 파일 영역으로 파일을 직접 드래그합니다.

USB 전송은 대형 라이브러리에 가장 빠르고 신뢰할 수 있는 방법으로, 수백 권을 몇 분 만에 전송합니다.

## 오프라인 시 읽기 진행 상황은 어떻게 됩니까?

BiblioFuse는 기기의 로컬 데이터베이스에 읽기 진행 상황을 기록합니다. 모든 페이지 넘김, 북마크, 주석이 먼저 로컬에 저장됩니다. 인터넷 연결이 가능해지면 변경 사항이 iCloud에 자동으로 동기화됩니다.

iPhone에서 오프라인으로 읽은 후 iPad나 Mac에서 BiblioFuse를 열면 진행 상황이 반영되어 있습니다. 동기화는 가장 최근 타임스탬프를 사용하므로 두 기기가 동시에 오프라인이었더라도 충돌이나 데이터 손실이 없습니다.

이 동기화를 시작하기 위해 아무것도 할 필요가 없습니다. BiblioFuse가 기기가 연결되는 즉시 백그라운드에서 처리합니다.

## 지원하는 오프라인 형식

BiblioFuse는 다음 형식을 완전히 오프라인으로 읽습니다:

| 형식 | 일반적인 용도 |
|------|------------|
| EPUB | 전자책, 소설, 논픽션 |
| CBZ | 만화 아카이브 (ZIP 기반) |
| CBR | 만화 아카이브 (RAR 기반) |
| ZIP | 이미지 아카이브 |
| PDF | 교재, 아트북, 스캔 문서 |

모든 형식이 온라인과 동일한 속도로 오프라인에서도 열립니다. 제한 모드, 워터마크, 광고가 없습니다.

## 자주 묻는 질문

**BiblioFuse를 오프라인으로 사용하려면 Apple 계정이 필요합니까?**

아니요. BiblioFuse는 Apple ID 없이 완전히 작동합니다. iCloud를 사용하지 않으면 진행 상황이 로컬에 저장되고 다른 기기와 동기화되지 않지만, 읽기, 북마크, 라이브러리 관리 등 다른 모든 기능은 정상적으로 작동합니다.

**Apple 외 곳에서 구입한 DRM 없는 전자책을 읽을 수 있습니까?**

네. BiblioFuse는 출처에 관계없이 DRM 없는 EPUB 파일을 읽습니다. 출판사, Project Gutenberg 또는 DRM 없는 EPUB 다운로드를 제공하는 상점의 전자책이 모두 작동합니다. BiblioFuse는 DRM으로 보호된 파일을 열 수 없습니다.

**파일이 기기 저장 공간을 차지합니까?**

네 — 파일이 로컬에 저장되므로 스트리밍 대신 기기 저장 공간을 사용합니다. 일반적인 EPUB 소설은 1~5 MB, 고품질 CBZ 만화는 80~200 MB입니다. 수십에서 수백 권 정도라면 현대 iPhone에서 저장 공간은 실질적인 문제가 되지 않습니다.

**비행기 모드에서도 앱이 작동합니까?**

완전히 작동합니다. 비행기 모드를 켜고 BiblioFuse를 열면 전체 라이브러리가 그대로 있고, 진행 상황 추적과 북마크가 정상적으로 작동합니다. iCloud 동기화는 재연결될 때까지 일시 중지됩니다.

## 핵심 정리

BiblioFuse는 "독서 생활이 서버 가용성에 의존해서는 안 된다"는 전제로 설계되었습니다. 파일은 로컬에 있고, 진행 상황은 먼저 로컬에 기록됩니다. 광섬유 연결을 사용하든 1만 미터 상공에 있든 독서 경험은 동일합니다.

전자책이나 만화 라이브러리를 보유하고 클라우드 의존 없이 iPhone에서 읽고 싶다면, [BiblioFuse](https://bibliofuse.com/ko/)가 직접적인 해답입니다. Wi-Fi, AirDrop, 또는 USB로 한 번 파일을 로드하면 — 어디서나 읽을 수 있습니다.
