# 如何在不使用云账户的情况下同步阅读进度

你正在 iPhone 上读一部漫画系列，拿起 iPad 却发现回到了第一页。云同步能解决这个问题——但前提是你愿意登录同步服务、把书库元数据托付给第三方，并接受网络抖动可能让进度出现偏差。

如果你不想这样呢？如果你出行时网络不稳定、不信任云服务存储阅读数据，或者就是想把一切保留在本地呢？

[BiblioFuse](https://bibliofuse.com/zh/) 提供了多种方式，让你无需依赖任何云账户就能在设备间保持阅读进度一致。以下是每种方案的具体说明。

## 为什么阅读进度同步比看起来更难

乍一看，"同步阅读进度"听起来很简单——不就是存一个数字吗？但漫画书库有几十乃至几百册，每册都有独立的总页数和最近阅读页码。阅读进度是一种**按书、按设备**的状态，可能在多台设备上同时发生变化。没有中央仲裁者（云服务器），两台同时离线的设备会产生冲突更新。

每种仅限本地的同步策略都有取舍：要么实行严格的主/副本模式，要么接受"最后写入的进度获胜"，要么只在设备处于同一网络时才同步。

BiblioFuse 的 iCloud 同步会自动处理仲裁。但如果 iCloud 不适合你，以下方案在不同自动化程度上给你掌控权。

## 方案一：通过 Mac 进行 Wi-Fi 书库同步（推荐用于家庭使用）

如果你的 Mac 运行了 BiblioFuse 的 Mac 主页书库功能，iPhone 可以通过 Wi-Fi 连接到它。Mac 同时作为文件和进度的权威来源。

**工作方式：**

1. 在 Mac 上打开 BiblioFuse，在设置 → 主页书库中启用书库服务器。
2. 在 iPhone 上，点击书库标签页中的网络图标，选择你的 Mac。
3. iPhone 显示 Mac 的书库，并直接从 Mac 存储中读取书籍——无需复制到 iPhone。

进度保存在 Mac 端。无论哪台设备读取 Mac 书库，进度都记录在 Mac 上。下次打开 iPhone 并重新连接 Mac 时，你会从上次 Mac 端阅读的位置继续。

**限制：** 需要两台设备在同一 Wi-Fi 网络下。最适合家庭使用；外出旅行时如需远程访问，可通过 VPN 或 Tailscale 实现。

## 方案二：手动导出与导入进度

BiblioFuse 允许将阅读进度导出为 JSON 文件，再导入到另一台设备。这是最便携、最适合离线使用的方案——无需任何网络。

**从设备 A 导出：**

1. 前往设置 → 书库 → 导出进度。
2. BiblioFuse 保存一个 `.json` 文件，包含书库中每本书最近阅读的页码。
3. 通过 AirDrop、文件 App 或其他方式分享该文件——无需网络。

**在设备 B 上导入：**

1. 收到 `.json` 文件后用 BiblioFuse 打开，或前往设置 → 书库 → 导入进度。
2. BiblioFuse 将导入的进度与本地现有进度合并，取每本书页码较大的那个（确保永远不会倒退）。

这种方法完全离线。设备相邻时 AirDrop 传输几乎即时完成。唯一的手动步骤是记住在切换设备前先导出——养成这个习惯只需几天。

## 方案三：通过本地网络硬盘或 USB 进行文件式同步

如果你通过文件 App 管理书库，进度可以随书籍文件一起传输。BiblioFuse 按文件名而非云端 ID 存储进度，这意味着：

- 将文件从设备 A 复制到设备 B（文件名相同），设备 B 上的 BiblioFuse 会显示无进度（文件对该设备而言是新的）。
- 但如果将书籍文件**和**进度导出文件（方案二）一起复制到设备 B，再导入进度文件，就能从正确页面继续阅读。

对于在 NAS 或 SMB 共享盘上管理书库的用户，这种方式运行简洁：书籍留在硬盘上，设备在局域网时连接访问，各设备独立追踪进度，手动同步进度文件即可。

## 方案四：不使用云同步账户的 iCloud Drive

这点值得澄清，因为经常造成混淆：**iCloud Drive 与 BiblioFuse 的 iCloud 同步不是同一回事。**

BiblioFuse 的 iCloud 同步使用 `NSUbiquitousKeyValueStore` 和 CloudKit 来同步阅读进度元数据，需要活跃的 iCloud 账户。

但你可以在不启用 BiblioFuse iCloud 进度同步的情况下，将**书籍文件**存储在 iCloud Drive（文件 App）中。这种配置下：

- 登录同一 Apple ID 的任意设备均可访问书籍文件
- 阅读进度在各设备本地独立追踪，**不会**同步
- 如需保持进度一致，仍需使用方案一、二或三

如果你只把 iCloud Drive 当作文件存储，不需要进度同步，这是一个简洁的设置。

## 方案五：接受各设备独立进度并使用书签

并非所有人都需要完美的进度同步。如果你主要一次读一本书，不在同一册中途切换设备，各设备独立进度完全够用。

BiblioFuse 允许在任意页面设置书签。书签是一个命名的保存位置——切换设备前创建一个书签，在另一台设备上立即跳转过去。设置一次点击，跳转也一次点击。对于偶尔切换设备的情况，这比完整同步更快。

## 选择适合你的方案

| 使用场景 | 最佳方案 |
|---------|---------|
| 家庭使用，有 Mac | Wi-Fi 主页书库（方案一） |
| 外出旅行，两台 iOS 设备 | AirDrop 手动导出/导入（方案二） |
| NAS 或共享盘书库 | 文件式同步 + 进度导出（方案三） |
| 偶尔切换设备，每次读一本 | 书签（方案五） |
| 有 Apple ID 但不用第三方云 | iCloud Drive 存文件 + 手动同步进度 |

## 隐私的角度

[BiblioFuse](https://bibliofuse.com/zh/) 将阅读进度存储在设备本地。禁用 iCloud 同步后，任何阅读数据都不会离开你的设备。对于不希望阅读习惯被任何外部服务记录的用户——无论出于隐私考虑还是个人偏好——仅本地模式是首选，而不是退而求其次的选项。

以上方案不是对缺失功能的变通。它们是在云同步不适合你的情况下使用 BiblioFuse 的设计方式。

## 开始使用

1. 从 App Store [下载 BiblioFuse](https://bibliofuse.com/zh/)。
2. 通过文件 App、Wi-Fi 传输或 Mac 主页书库导入漫画或电子书文件。
3. 选择适合你阅读习惯的同步策略——也可以组合使用。方案二和方案五可以与任何其他方案并行使用。

退出云账户不必牺牲便利。养成一个小习惯——切换设备前快速 AirDrop 导出——就能让书库状态保持一致，无需把阅读数据托付给任何人。
