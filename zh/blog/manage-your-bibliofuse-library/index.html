管理你的 BiblioFuse 书库：文件夹、传输与 Mac 流式访问

BiblioFuse 在 iPhone、iPad 和 Mac 上管理您的 EPUB、CBZ、CBR、PDF 和 TXT 书库，不会移动或锁定您的文件。在 iOS 17 或更高版本上，您可以通过 Wi-Fi 传输、iCloud 云盘或文件 App 的分享表单加载文件，并通过 Wi-Fi 将整个 Mac 书库流式传输到 iPhone，无需占用设备存储空间。BiblioFuse 不会重命名文件或将其锁入任何专有格式。

BiblioFuse 建立在一个简单的原则上：你的文件留在你放置的地方。应用永远不会移动、重命名或将文件锁入专有格式。理解书籍存放在哪里——以及如何将它们放到那里——决定了书库是为你服务，还是让你不断与它较劲。

如何添加书籍：四种方式

Mac 和 Android——添加文件夹

在 Mac 和 Android 上，BiblioFuse 监视文件夹，而不是接管文件。在侧边栏中点击添加文件夹，指向任意位置——下载文件夹、外置硬盘、NAS 都可以。BiblioFuse 会索引内容并追踪变化，不会碰文件本身。

iOS——复制到 BiblioFuse 文件夹

iPhone 和 iPad 使用沙箱文件模型，因此需要将书籍复制到应用可以看到的位置。有两个可用位置：

iCloud Drive → BiblioFuse — 打开"文件"应用，导航到 iCloud 云盘 → BiblioFuse。把你的电子书文件夹复制或移动到这里。每个放入的文件夹都会在 BiblioFuse 的书目列表中显示为一个独立条目。直接放在 iCloud 云盘 → BiblioFuse 根目录（不在任何子文件夹中）的书籍或文件，在应用中会显示在名为 iCloud 的文件夹下。存储在此处的书籍会自动同步阅读进度：你的阅读位置在 iPhone、iPad 和 Mac 之间同步。在任意设备上，都能从上次停止的地方继续。

我的 iPhone → BiblioFuse — 若只需本地存储，将电子书文件夹复制到这里。文件夹以同样方式显示在列表中，但进度只保存在该设备上。

两种方式都请放置文件夹——而不是单个文件。一个包含漫画系列各卷的文件夹，在列表中显示为一个条目，所有卷都在里面。

Wi-Fi 传输

内置的浏览器导入服务器让你无需数据线或同步软件，就能从任何电脑拖入文件。

  • iOS：前往设置 → Wi-Fi 导入并启用。屏幕上会显示一个本地 URL——在桌面浏览器中打开它，然后拖入文件。文件会进入我的 iPhone → BiblioFuse → import，并自动出现在书库中。
  • Android：配置 Wi-Fi 传输时，你可以选择目标文件夹。

工具处理过的书籍

使用工具标签页压缩或合并文件后，输出会到：

  • iOS：我的 iPhone → BiblioFuse → export — 处理完的文件会自动出现在这里。
  • Android：运行工具时，你可以选择输出文件夹。

BiblioFuse 网页工具在任何浏览器中提供相同的功能——在 Windows 或 Linux 上特别方便。关于压缩节省空间的详细说明，请参阅我们的漫画和电子书文件大小指南。

Mac：右键单击即可发送到 iCloud

如果你正在 Mac 上阅读，想把某本书同步到 iPhone，右键单击书籍封面，选择发送到 iCloud。BiblioFuse 会将文件直接复制到 iCloud 云盘 / BiblioFuse——这是你 iPhone 正在监视的同一个容器。几分钟内就会出现在手机上，无需任何手动操作。

要一次发送多本书，启用选择模式（右上角的勾选图标），选好书籍，然后点击操作栏中的发送到 iCloud。

书库网格：长按选项

在 iOS 上长按任意书籍封面，会弹出上下文菜单：

  • 打开 — 直接进入阅读
  • 分享 — 将文件传递给其他应用
  • 重命名 — 更改显示名称（不会重命名磁盘上的文件）
  • 编辑元数据 — 添加标签和 1 到 5 星评分
  • 移除 — 出现确认框，有两个选项：
    • 从书库移除 — 删除书库条目，文件在磁盘上保持不变
    • 从存储删除 — 同时删除条目和物理文件

要将书籍发送到工具标签页，进入选择模式（长按第一本书，然后点击其他书籍）。底部会出现一个带工具按钮（魔棒图标）的工具栏。点击它，将所选书籍加入压缩或合并队列。

从 Mac 流式访问书库

如果你的主要藏书在 Mac 上，不需要复制任何内容。BiblioFuse 可以直接从 Mac 流式传输到 iPhone——但请使用 Tailscale，而非基础 Wi-Fi 方式，因为后者在手机切换网络时可能不稳定。

Tailscale 在你的设备之间建立私有加密网络。配置完成后，无论你在使用移动数据、公共 Wi-Fi 还是家庭网络，iPhone 都能访问你的 Mac。

第一步：在两台设备上安装 Tailscale

  1. Mac：从 tailscale.com/download 或 Mac App Store 下载。
  2. iPhone：在 App Store 搜索"Tailscale"。
  3. 在两台设备上用同一个 Tailscale 账户登录。

连接后，两台设备就在同一个私有网络中，可以互相访问。

第二步：在 Mac 上启用流式传输

在 Mac 版 BiblioFuse 中，向下滚动左侧边栏到流式传输部分，打开开关。会出现两个端点：一个用于局域网/Wi-Fi，另一个用于远程 / Tailscale。Tailscale 是可靠的选择。

第三步：检查 iPhone 上的权限

需要两个 iOS 权限：

  1. 本地网络 — 前往设置 → 隐私与安全性 → 本地网络，开启 BiblioFuse。
  2. iCloud — 前往设置 → [你的姓名] → iCloud，开启 BiblioFuse。

快速检查：打开 iPhone 上的 BiblioFuse，前往设置 → 权限。绿色对勾表示一切正常；红色标记会直接跳转到需要修改的系统设置。

第四步：从 iPhone 连接

  1. 在 BiblioFuse 中打开书库标签页。
  2. 在侧边栏（iPad 上是左侧面板，iPhone 上点击侧边栏图标），向下滚动到底部——Mac 部分列出了已发现的流式端点。
  3. 找到名称中带有 (iCloud) 的条目，例如 Matt 的 MacBook Pro (iCloud)。
  4. 点击它。等待 10–30 秒，建立安全连接。

Mac 书库文件夹现在出现在侧边栏中，就像本地存储一样。正常浏览和阅读即可。

流式传输 vs. 下载

格式行为方式
CBZ / CBR逐页流式传输——无需下载，立即打开
EPUB / TXT先下载到设备，再打开

阅读进度说明： 流式会话的进度保存在 Mac 的本地数据库中，而非 iCloud。不会同步到其他设备——进度留在 Mac 上。

快速提示

  • iCloud 是最省力的同步方式：存放在 iCloud 云盘 → BiblioFuse 中的书籍，会自动在所有 Apple 设备上同步并追踪进度。
  • export 文件夹是暂存区：在 iOS 上压缩后，在我的 iPhone → BiblioFuse → export 中找到结果，再移动到长期存放的位置。
  • 流式连接不通先检查权限：大多数连接问题都源于本地网络或 iCloud 未开启。

常见问题

BiblioFuse 在 iPhone 上将文件保存在哪里？ BiblioFuse 使用两个在"文件"应用中可见的文件夹：iCloud 云盘 → BiblioFuse（跨设备同步）和我的 iPhone → BiblioFuse（仅限本地）。Wi-Fi 传输的文件进入本地 import 子文件夹；通过 iCloud 添加的文件进入 iCloud 文件夹。

如何无需数据线将书籍从 Mac 传输到 iPhone？ 使用 Wi-Fi 传输：在 iPhone 的 BiblioFuse 中前往设置 → Wi-Fi 导入并开启，然后在 Mac 浏览器中访问显示的 URL 并拖入文件。您也可以将文件放入 Mac 上的 iCloud 云盘 → BiblioFuse，它们会自动同步。

BiblioFuse 支持基于文件夹的系列整理吗？ 支持。将一个 CBZ 卷册文件夹（例如漫画系列）放入 BiblioFuse 文件夹——每个子文件夹在书库中显示为独立的系列条目。文件夹内的各卷按阅读顺序列出。

Mac Home Library 流式传输与 iCloud 同步有什么区别？ iCloud 同步将文件复制到设备，可在任何地方离线阅读。Mac Home Library 通过 Wi-Fi 按需从 Mac 流式传输页面——不会复制任何内容到 iPhone，因此不占用设备存储，但需要在同一网络（或使用 Tailscale 远程访问）。

如何快速将 Mac 书库中的书籍发送到 iPhone？ 在 Mac 版 BiblioFuse 中右键单击任意书籍封面，选择"发送到 iCloud"。BiblioFuse 将文件复制到 iCloud 云盘 → BiblioFuse，几分钟内即可在 iPhone 上看到。
