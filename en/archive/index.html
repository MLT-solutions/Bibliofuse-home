FAQ

Frequently Asked Questions

What is perceptual hashing and why does it matter?

Perceptual hashing compares the visual structure of images rather than raw file bytes. Two files can have different names, sizes, and metadata — but if their content looks the same, perceptual hashing matches them. Standard duplicate finders fail the moment a file is renamed or re-exported; Comic Duplicate Scanner does not.

Can it find duplicates that have been renamed?

Yes. Renaming a file changes nothing about its content. Comic Duplicate Scanner compares the images inside each archive, so a duplicate is detected regardless of what the file is called.

Is deletion permanent?

No. Marked files are moved to the system Trash (macOS) or Recycle Bin (Windows), not permanently deleted. You can recover any file before emptying the Trash.

Which file formats are supported?

Comic archives: CBZ, CBR, ZIP, RAR. Photos: JPG, PNG, HEIC, WEBP, BMP, GIF. Filename fuzzy matching works with any file type.

Does it work with photos, not just comics?

Yes. Select Photos mode to scan JPG, PNG, HEIC, WEBP, BMP, and GIF files. It works with iPhone photo library exports, scanned documents, and digital art collections.

Is there a trial before I buy?

You can scan up to 1,000 files for free, forever. A 7-day free trial on Mac App Store unlocks unlimited access, and the one-time Pro purchase is $19.99 with no subscription.

Can it match a color scan with a black-and-white version of the same content?

Yes. The dHash algorithm works on luminance gradients, not color values. A full-color scan and a grayscale version of the same page produce nearly identical hashes and will be flagged as duplicates.

Does image resolution affect duplicate detection?

No. dHash resizes every image to a small fixed grid before comparing, so a 1200px and a 3000px scan of the same page hash to the same value. Resolution differences between sources are completely ignored.
